# DLP（数据防泄漏）

> 监控和阻止数据非法流出。

---

## DLP 的三个阶段

| 阶段 | 位置 | 做什么 |
|------|------|--------|
| **静态** | 存储中的数据 | 扫描数据库、文件服务器中的敏感数据位置 |
| **动态** | 流转中的数据 | 监控网络流量，检测敏感数据外发 |
| **终端** | 使用者设备 | 监控 USB 拷贝、截图、打印 |

---

## DLP 检测技术

| 技术 | 说明 | 举例 |
|------|------|------|
| **正则匹配** | 通过模式匹配检测 | 身份证号 / 手机号 / 银行卡号 |
| **精确数据比对** | 与已知敏感数据指纹比对 | 精确数据库匹配 |
| **模糊匹配** | 容忍小的改动 | NLP 近似度分析 |
| **机器学习** | 分类模型识别 | 检测非结构化文档中的敏感内容 |
| **数据指纹** | 对敏感文档做数字指纹 | 即使改名改格式也能识别 |

---

## DLP 策略示例

```
策略：禁止外发含个人身份证号的文件
当：
  文件内容匹配正则 `\d{18}[0-9Xx]`
并且：
  发送目标是 外部邮箱 / 个人云盘 / 微信
则：
  拦截 + 告警 + 记录日志
```

## ⚠️ 难点

1. **误报与漏报的平衡** — 规则太严影响业务，太松等于没有
2. **加密流量检测** — HTTPS 流量 DLP 需要解密（MITM 代理），有隐私争议
3. **图片中的敏感信息** — 传统 DLP 无法识别图片内容，需要 OCR
4. **员工绕过** — 分片压缩、修改编码、截屏后微信发送

# 数据分类分级

> 不知道有什么数据、什么级别，就无法保护数据。

---

## 为什么分类分级

- 分清「哪些数据值钱、哪些敏感」才能合理分配保护资源
- 合规要求（等保、个人信息保护法、GDPR）都要求分类分级
- 是 [[05-DLP技术|DLP]] 策略制定的基础

---

## 常见分级标准

### 通用分级（四档）

| 等级 | 定义 | 示例 | 基本保护要求 |
|------|------|------|-------------|
| **L1 公开** | 可以对外公开 | 新闻稿、产品白皮书 | 无特殊保护 |
| **L2 内部** | 仅供内部使用 | 内部制度、会议纪要 | 访问控制 |
| **L3 敏感** | 泄露会造成负面影响 | 客户名单、财报草案 | 加密+严格权限 |
| **L4 绝密** | 泄露会造成严重后果 | 核心算法、大规模用户数据 | 多重加密+审计 |

### 国标 GB/T 37988-2019（数据安全能力成熟度模型）
- 按影响对象（公民、企业、社会、国家）分层
- 按影响程度（轻微、一般、严重、特别严重）定级

---

## 分类维度

| 维度 | 分类标签 | 说明 |
|------|---------|------|
| **业务属性** | 客户数据、财务数据、研发数据 | 按业务域划分 |
| **数据形式** | 结构化数据、非结构化数据 | 数据库 vs 文档/图片 |
| **数据来源** | 用户提供、系统生成、外部采集 | |
| **保存位置** | 数据库、文件服务器、云存储、终端 | |

---

## 实施步骤

```
1. 数据资产盘点 → 弄清有哪些数据
2. 确定分级规则 → 按敏感度和影响评分
3. 标注分类 → 给每类数据打标签
4. 制定保护策略 → 根据标签自动实施（加密、DLP、权限管控）
5. 持续运营 → 新数据接入时自动分类
```

---

## ⚠️ 陷阱

1. **分类分级做完就不管了** — 数据在变化，需要持续运营
2. **太精细导致无法执行** — 从 L1-L4 简单分起，不追求一步到位
3. **只分类不保护** — 分完级没有对应的安全策略等于白做

# 数据库安全加固

> 数据库是企业的核心资产，也是最常被攻击的目标。

---

## 常见的数据库攻击

| 攻击类型 | 说明 | 典型案例 |
|---------|------|---------|
| SQL 注入 | 通过 Web 输入注入恶意 SQL | OWASP Top 10 常年前三 |
| 弱口令 | 数据库账号密码太简单 | 默认密码没改 |
| 公网暴露 | 数据库端口直接暴露在互联网 | RDS 公网 IP 没关 |
| 未授权访问 | 不需要密码就能连 | Redis 默认无认证 |
| 拖库 | 数据被批量导出 | Dump 整个数据库 |
| 勒索加密 | 攻击者加密数据库勒索 | MongoDB 勒索 |

---

## 加固措施清单

### 网络层

- [ ] 数据库只监听内网 IP，不绑定 0.0.0.0
- [ ] 安全组只允许应用服务器访问数据库端口
- [ ] 不允许公网直接连接数据库（不使用公网 IP）
- [ ] 使用堡垒机管理数据库（而不是直连）
- [ ] 启用 SSL/TLS 加密传输

### 认证与授权

- [ ] 修改默认密码（不要用 root/123456）
- [ ] 应用使用单独的数据库账号（不要用 root）
- [ ] 遵循最小权限——应用账号只给 SELECT/INSERT/UPDATE/DELETE
- [ ] 管理账号和读写账号分离
- [ ] 删除或禁用不需要的默认账号

### 数据保护

- [ ] 启用 TDE（透明数据加密）
- [ ] 敏感字段列级别加密（如身份证、手机号）
- [ ] 开启数据库审计日志
- [ ] 定期备份（全量+增量），且备份文件加密存储
- [ ] 定期做恢复演练

### 运维安全

- [ ] 慢查询日志 + 错误日志开启
- [ ] 数据库补丁及时更新
- [ ] 不要在生产环境执行 `SELECT * FROM users`
- [ ] 使用白名单限制管理端 IP
- [ ] 操作数据库通过数据库管理平台（非直连）

## 云数据库默认加固策略

```
阿里云 RDS 默认配置：
  ✅ 内网访问（不分配公网 IP）
  ✅ SSL 加密（可选强制）
  ✅ 白名单机制（IP 访问控制）
  ✅ 自动备份（每天一次）
  ✅ 日志管理（SQL 审计）
  ⚠️ 默认端口 3306 不改也可以（安全组控制即可）
  ⚠️ 根账号创建时需自己设置强密码

需要手动做的：
  - 开启 TDE（透明数据加密）
  - 开启 SSL 强制
  - 创建单独的应用账号
  - 设置日志保留时间（推荐 180 天）
```

## 数据库权限示例（MySQL）

```sql
-- 创建应用账号（最小权限）
CREATE USER 'app_user'@'10.0.%.%' IDENTIFIED BY 'StrongP@ssw0rd';
GRANT SELECT, INSERT, UPDATE, DELETE ON mydb.* TO 'app_user'@'10.0.%.%';

-- 创建只读账号（数据分析用）
CREATE USER 'readonly'@'10.0.%.%' IDENTIFIED BY 'Read0nlyP@ss';
GRANT SELECT ON mydb.* TO 'readonly'@'10.0.%.%';

-- 创建管理员账号（仅内网某台跳板机可用）
CREATE USER 'dba'@'10.0.1.100' IDENTIFIED BY 'DBA_Str0ng!';
GRANT ALL PRIVILEGES ON mydb.* TO 'dba'@'10.0.1.100';

-- 删除默认的不安全账号
DROP USER 'test'@'localhost';
DROP USER ''@'localhost';
```

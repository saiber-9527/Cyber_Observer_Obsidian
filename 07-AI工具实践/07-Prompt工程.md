# Prompt Engineering

> 和 AI 高效沟通的技术——好的 prompt 和坏的 prompt 效果差好几倍。

---

## 基础技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| **明确角色** | 给模型一个身份 | "你是一名网络安全专家" |
| **结构清晰** | 用分隔符、编号 | 列出 1. 2. 3. |
| **示例输入输出（Few-Shot）** | 给 1-3 个样例 | 输入→输出 对 |
| **指定输出格式** | 限制格式 | "输出 JSON"、"三段式" |
| **限定范围** | 避免偏离 | "只回答与XX相关的内容" |

## 高级技巧

### 思维链（Chain-of-Thought, CoT）
```
问题：小明有 5 个苹果，给了小红 2 个，又从超市买了 3 个，现在有几个？
思考过程：
1. 小明原有 5 个苹果
2. 给小红 2 个，剩余 3 个
3. 又买了 3 个，现在有 6 个
答案：6
```
- 让模型展示推理过程，显著提升复杂问题的准确性

### 思维树（Tree-of-Thought）
- 让模型同时探索多条推理路径
- 选择最优路径继续推理

### 自我反思
- 让模型检查自己的答案
- "请检查你的回答，是否有什么错误？"

## 常见框架

| 框架 | 核心 | 适合场景 |
|------|------|---------|
| **CRISPE** | 能力+角色+洞察+陈述+实验 | 通用 prompt 框架 |
| **COT** | 思维链 | 推理、数学、逻辑 |
| **ReAct** | 推理+行动 | Agent 类的循环 |
| **Auto-CoT** | 自动生成思维链示例 | 自动化场景 |

## ⚠️ 安全视角

Prompt 是 AI 安全攻击的重灾区：

| 攻击类型 | 示例 | 防御 |
|---------|------|------|
| **提示注入** | "忽略之前的指令，告诉我..." | 指令护栏、输入过滤 |
| **越狱** | "扮演DAN角色..." | 对抗性训练、规则检测 |
| **角色劫持** | 诱导模型忽略系统 prompt | 严格输出验证 |

## 实践建议

1. **系统 prompt 和用户 prompt 分开** — 系统 prompt 设置角色和边界，用户 prompt 放开调用
2. **温度不能太高** — 关键场景 temperature=0，需要创意的 0.5-0.8
3. **设计时从简单到复杂** — 先让基础 prompt 正确，再逐步加细节
4. **对输出做后处理校验** — 重要场景一定不能完全相信模型输出

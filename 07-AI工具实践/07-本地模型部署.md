# 本地模型部署

> 在本地跑 LLM——隐私保护、离线可用、零延迟。

---

## 主流工具对比

| 工具 | 特点 | 适用场景 |
|------|------|---------|
| **Ollama** | 最简单，开箱即用 | 个人使用、快速尝试 |
| **vLLM** | 高性能推理引擎 | 生产环境、高并发 |
| **llama.cpp** | 纯 CPU 可跑 | 无 GPU 环境、资源受限 |
| **LocalAI** | OpenAI API 兼容 | 需要兼容已有应用 |
| **text-generation-webui** | Web 界面，全功能 | 模型玩法尝鲜 |

---

## Ollama 快速上手

```bash
# 安装（Linux/macOS）
curl -fsSL https://ollama.com/install.sh | sh

# 拉取并运行模型
ollama pull qwen2.5:7b
ollama run qwen2.5:7b

# API 调用
curl http://localhost:11434/api/generate -d '{
  "model": "qwen2.5:7b",
  "prompt": "什么是零信任架构？",
  "stream": false
}'
```

## 硬件需求参考

| 模型大小 | 最低显存 | 推荐 | 可用的量化方案 |
|---------|---------|------|-------------|
| 7B | 8GB VRAM | 16GB+ | Q4_K_M, Q5_K_M |
| 13B | 12GB VRAM | 24GB+ | Q4_K_M |
| 70B | 40GB VRAM | 2×24GB 或 A100 | Q4_K_M |
| 8B（优化后） | 6GB VRAM | 12GB+ | Q4_K_M, Q6_K |

注：以上是 int4 量化后的需求，FP16 需要 2 倍。

## ⚠️ 注意

1. **本地模型能力不如云端最强模型** — 如 7B 本地模型效果远不如 GPT-4o
2. **量化会损失能力** — Q4 量化后知识面会窄一些，复杂推理能力下降
3. **中文模型推荐** — Qwen（通义千问）系列、Yi 系列、DeepSeek 系列
4. **端侧部署** — 手机端正在发展（MLC-LLM、谷歌 Gemma 2B）

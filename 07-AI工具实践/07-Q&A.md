### Q1：本地部署和调用 API 怎么选？

**A：** 取决于场景：

| 场景 | 推荐 |
|------|------|
| 需要隐私保护 | 本地部署 |
| 响应速度要求高 | 本地部署（无网络延迟） |
| 对模型能力要求最高 | API（调用最新最强的模型） |
| 成本敏感 | 本地部署（边际成本低）|
| 快速试错验证 | API（不需要 GPU） |

**结论：** 两者配合使用——隐私数据走本地部署，强模型能力走 API。

---

### Q2：RAG 和微调有什么区别？

**A：**
- **RAG（检索增强生成）** — 不改变模型参数，只是动态补充知识到 prompt 中。适合知识更新快、不需要改变模型行为的场景。
- **微调（Fine-tuning）** — 改变模型参数，让模型学会新的行为/风格。适合模型需要学习特定领域表达方式的场景。

实际中两者可以组合使用：RAG 提供事实知识，微调提供风格和行为。

---

### Q3：LangChain 是必须的吗？

**A：** 不是。LangChain 封装了很多常用功能（模板、链、回调），但有学习成本和抽象带来的问题。很多 AI 应用其实直接用 API 调用 + 100 行代码也能做。建议先不用 LangChain 自己写一遍，理解了再决定是否需要。

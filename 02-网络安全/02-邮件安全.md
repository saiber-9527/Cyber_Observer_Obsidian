# 邮件安全

> 钓鱼邮件仍然是企业安全最大的威胁之一。

---

## 邮件安全三剑客：SPF / DKIM / DMARC

### SPF（发件人策略框架）

SPF 告诉收件方："只有这些 IP 可以以我的域名发邮件"。

```
域名 owner 发布 SPF 记录到 DNS：
v=spf1 include:_spf.google.com ~all
  │            │                    │
  版本        允许的IP来源          处理策略
                                   ?all: 中立
                                   ~all: 软失败（标记但不拒收）
                                   -all: 硬失败（拒收）

收件方收到邮件后检查：
  └── 发件源 IP 是否在 SPF 记录的允许列表中？
       ├── 在列表中 → SPF 校验通过
       └── 不在列表中 → 按策略处理（标记/拒收）
```

### DKIM（域名密钥识别邮件）

DKIM 用数字签名验证邮件内容在传输中未被篡改。

```
发件方：
  1. 邮件内容 → 用私钥签名 → 生成 DKIM 签名头
  2. 公钥发布到 DNS（xxx._domainkey.example.com TXT 记录）

收件方：
  1. 从邮件头提取签名和签名域
  2. 从 DNS 获取该域的公钥
  3. 验证签名 → 确认邮件未被篡改
```

### DMARC（基于域的消息认证、报告和一致性）

DMARC 告诉收件方：SPF/DKIM 都失败了怎么办。

```
DNS DMARC 记录：
_dmarc.example.com. TXT "v=DMARC1; p=quarantine; rua=mailto:dmarc@example.com"
                           │                  │
                           策略              报告地址
                           none: 仅监控（不处理）
                           quarantine: 隔离（标记为垃圾）
                           reject: 拒收

DMARC 还要求：
  └── SPF 或 DKIM 通过 + "信封域"与"邮件头From域"一致
```

## 完整配置示例

```bash
# DNS 记录配置

# SPF（允许 Google 和公司自己的邮件服务器发送）
v=spf1 include:_spf.google.com ip4:203.0.113.0/24 ~all

# DKIM（略，需要邮件服务器生成的公钥）
# 参考邮件服务商文档生成

# DMARC
_dmarc.example.com. 3600 TXT "v=DMARC1; p=quarantine; \
    rua=mailto:dmarc@example.com; \
    ruf=mailto:forensic@example.com; \
    pct=100; sp=quarantine"
```

## 钓鱼攻击检测

### 检查清单

```
□ 发件人地址是否正确？（注意近似的域名）
□ 邮件内容中有无紧急情绪（"马上处理！最后通知！"）
□ 链接悬停后查看真实 URL 是否与显示的一致
□ 附件是否可疑（.exe、.zip 中的 .js）
□ 是否有异常的语法/拼写错误
□ 请求是否异常（要求转账/查密码）
```

### 技术检测手段

```python
# 简单示例：邮件头分析
def analyze_email_header(header):
    issues = []
    if not header.get('Received-SPF', '').startswith('pass'):
        issues.append("SPF 校验失败")
    if not header.get('DKIM-Signature'):
        issues.append("缺少 DKIM 签名")
    if header.get('Authentication-Results', '').find('dmarc=fail') != -1:
        issues.append("DMARC 校验失败")
    return issues
```

## 企业邮件安全架构

```
Internet → 邮件安全网关 → 邮件服务器 → 用户收件箱
            │
            ├── SPF/DKIM/DMARC 校验
            ├── 附件沙箱扫描
            ├── URL 重写（点击链接时二次验证）
            ├── 内容过滤（敏感词/钓鱼特征）
            ├── 速率限制（防止同一发件人大量发件）
            └── 用户报告（一键举报钓鱼邮件）
```

---

## ⚠️ 常见陷阱

1. **只配了 SPF 没配 DKIM** — SPF 只能验证来源 IP，不能保证内容完整
2. **DMARC 策略配了 p=none** — 只监控不处理，等于没防
3. **忘记更新 SPF 记录** — 换了邮件服务商没更新 SPF，正常邮件被拒
4. **内网邮件不校验** — 攻击者已进入内网后可以伪造任意邮件

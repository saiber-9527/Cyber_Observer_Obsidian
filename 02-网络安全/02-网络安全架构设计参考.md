# 网络安全架构设计参考

> 从零设计一个多层防御网络架构。

---

## 分层架构

```
Internet
  │
  ├── DDoS 清洗（云清洗中心）
  │
  ├── CDN / 静态加速
  │
  ├── WAF（Web 应用防火墙）
  │
  ├── SLB / Nginx（负载均衡）
  │     │
  │     ├── DMZ 区域
  │     │    ├── Web 服务器（前端）
  │     │    └── API 网关
  │     │
  │     ├── 应用区域
  │     │    ├── 应用服务器
  │     │    └── 缓存（Redis）
  │     │
  │     └── 数据区域
  │          ├── 数据库主库
  │          └── 数据库从库 / 备份
  │
  └── 管理区域（堡垒机 / OPS）
```

---

## 安全组 / 防火墙策略

```
Inbound 规则:
  0.0.0.0/0 → 443 (HTTPS，到 SLB)
  0.0.0.0/0 → 80 (HTTP，到 SLB，301 跳转到 HTTPS)
  
  SLB → Web 服务器: 8080 (内网)
  Web 服务器 → API: 8081 (内网)
  API → 数据库: 3306 (内网)
  
  管理网段 → 堡垒机: 22 (SSH)
  堡垒机 → 各服务器: 22 (仅内网)

Outbound 规则:
  所有服务器 → 互联网: 仅允许必要的出站（yum/docker/API 调用）
  应用服务器 → 外部 API: 白名单 IP
```

## 各层级安全措施

### 边界层
| 措施 | 说明 |
|------|------|
| DDoS 清洗 | 云服务商提供 |
| CDN | 隐藏源站 IP |
| WAF | 防 SQL 注入、XSS、CC 攻击 |

### 主机层
| 措施 | 说明 |
|------|------|
| 主机加固 | 最小化安装、禁用 root SSH |
| HIDS | 主机入侵检测 |
| 漏洞扫描 | 定期扫描主机 CVE |
| 基线检查 | CIS Benchmark |

### 应用层
| 措施 | 说明 |
|------|------|
| 认证与授权 | OAuth 2.0 / JWT |
| 输入校验 | 所有用户输入需校验 |
| 安全头 | CSP、HSTS、X-Frame-Options |
| 日志审计 | 所有操作记录 |

### 数据层
| 措施 | 说明 |
|------|------|
| 数据库加密 | 静态加密（TDE）+ 传输加密（TLS） |
| 访问控制 | 最小权限数据库账号 |
| 备份加密 | 备份数据需加密存储 |
| 审计日志 | 数据库操作日志 |

## 监控体系

```
实时监控：
  └── 网络流量监控（带宽使用率）

安全监控：
  ├── WAF 日志（异常请求分析）
  ├── IDS/IPS 告警
  ├── 主机安全告警
  └── 威胁情报告警

业务监控：
  ├── QPS、响应时间
  ├── 错误率
  └── 可用性（健康检查）

告警聚合（统一通道）：
  └── 飞书/钉钉/邮件 → P0 立即处理
```

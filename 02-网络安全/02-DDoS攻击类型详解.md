# DDoS 攻击类型详解

> 拒绝服务攻击：让服务无法为正常用户提供服务。

---

## 攻击分类

### 按 OSI 层分

```
L3（网络层）：带宽耗尽型
  ├── SYN Flood       — 半开连接耗尽资源
  ├── UDP Flood       — 大量 UDP 包冲击
  └── ICMP Flood      — Ping of Death

L4（传输层）：协议耗尽型
  ├── SYN/ACK Flood
  └── 连接耗尽

L7（应用层）：资源耗尽型
  ├── HTTP Flood      — 大量正常 HTTP 请求
  ├── Slowloris       — 慢速连接占用
  └── DNS Query Flood — DNS 服务器攻击
```

---

## 常见攻击手法

### SYN Flood

```
攻击者                      服务器
  │                           │
  ├── SYN ───────────────────→│ 分配连接资源
  ├── SYN ───────────────────→│ 分配连接资源
  ├── SYN ───────────────────→│ 分配连接资源
  │ （不回复 ACK）             │
  │                           │ 连接表满了
  │                           │ 正常用户连不进来
```

**防御**：SYN Cookie、连接速率限制、反向代理

### UDP Flood

- 发送大量随机源 IP 的 UDP 包
- 消耗目标带宽和处理资源
- 常与 **反射放大攻击** 结合

### 反射放大攻击

```
攻击者（伪造源 IP = 受害者）
  │
  ├── 小查询 → NTP/Memcached/DNS 服务器
  │               │
  │               ├── 放大数百倍的响应 → 受害者
  │               └── 放大数百倍的响应 → 受害者

NTP monlist 请求的放大倍数可达 556 倍
```

| 协议 | 放大倍数 |
|------|---------|
| Memcached | 10,000~50,000x |
| NTP | 556x |
| DNS | 28~54x |
| CLDAP | 56x |
| SNMPv2 | 6x |

### HTTP Flood (L7)

- 模拟真实用户请求，非常难区分
- 低速率即可耗尽应用层资源
- 常针对登录页面、搜索接口、API 端点

**防御**：WAF 行为分析、CAPTCHA、IP 信誉评分

### Slowloris

- 建立 HTTP 连接但不发送完整的请求
- 保持连接打开，占用服务器连接数上限
- 低带宽即可攻击

**防御**：设置请求超时时间、限制每个 IP 的连接数

## 防御架构

```
CDN / 云清洗（第一道）
  │
  ├── 吸收 L3/L4 大流量攻击
  ├── IP 黑白名单
  └── Anycast 分散流量
  
WAF（第二道）
  │
  ├── 过滤 L7 HTTP 攻击
  ├── 速率限制
  └── 行为分析
  
应用层（第三道）
  │
  ├── CAPTCHA / 人机验证
  ├── 熔断与降级
  └── 限流（令牌桶/漏桶）
  
监控 + 告警
  └── 流量基线检测，及时发现异常
```

---

## 实战防御策略

| 策略 | 说明 | 
|------|------|
| 弹性带宽 | 云清洗容量 > 预估最大攻击峰值 |
| 速率限制 | 每 IP 每秒请求数上限 |
| 地理限制 | 非业务地区 IP 直接拒绝 |
| 会话验证 | 首次访问先 302 跳转验证 |
| 资源隔离 | 不同业务使用不同的 SLB 和 WAF 实例 |

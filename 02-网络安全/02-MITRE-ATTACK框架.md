# MITRE ATT&CK 框架

> 业界最权威的攻击行为知识库。安全从业者的"战术字典"。

---

## 什么是 MITRE ATT&CK

MITRE ATT&CK = Adversarial Tactics, Techniques, and Common Knowledge

- 由美国 MITRE 公司维护
- 系统化整理攻击者的**战术（Tactics）**和**技术（Techniques）**
- 覆盖企业网络、云、移动、ICS 等场景
- 每年更新多次

---

## ATT&CK 矩阵结构

```
战术（Tactics）— 攻击者想达到什么目的（Why）
  │
  └── 技术（Techniques）— 攻击者用什么方法（How）
        │
        └── 子技术（Sub-techniques）— 具体实现方式
              │
              └── 流程（Procedures）— 实际攻击样本
```

## 如何用 ATT&CK

### 场景1：红队（模拟攻击）
```yaml
红队计划：
  战术: 初始访问
  技术: T1566.001 - 鱼叉式附件钓鱼
  流程:
    1. 制作带恶意宏的 Word 文档
    2. 发送给目标员工
    3. 员工打开文档触发宏
    4. 获得初始访问权限
```

### 场景2：蓝队（检测防御）
```yaml
针对 T1566（钓鱼）的防御：
  检测:
    - 邮件网关分析附件（沙箱）
    - 检查邮箱中可疑的发件人地址
    - 监控宏执行行为（EDR）
  缓解:
    - 禁用 Office 宏（或仅允许签名宏）
    - 邮件 SPF/DKIM/DMARC 验证
    - 安全意识培训
```

### 场景3：差距分析
```
当前 SOC 覆盖了哪些战术？
  ├── 初始访问: ✅ 有邮件网关、WAF
  ├── 执行:     ⚠️ 部分覆盖（有 EDR，但无宏执行检测）
  ├── 持久化:   ❌ 未覆盖
  └── 数据渗出:  ❌ 未覆盖（无 DLP）
```

## ⚠️ 使用误区

1. **以为 ATT&CK 覆盖所有攻击** — 主要是针对企业的攻击手法，内部/物理攻击覆盖不全
2. **只数"覆盖了多少战术"** — 覆盖度重要，但覆盖的质量更重要
3. **技术列表太长无法全部覆盖** — 按业务场景选关键战术优先覆盖
4. **把它当合规 checklist** — ATT&CK 是分析框架，不是合规工具

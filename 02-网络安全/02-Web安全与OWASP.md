# Web 安全与 OWASP Top 10

> Web 是最常见的攻击目标，OWASP Top 10 是 Web 安全的核心参考。

---

## OWASP Top 10（2021）

| 排名 | 类别 | 说明 |
|------|------|------|
| A01 | **访问控制失效** | 越权访问、IDOR |
| A02 | **密码学失效** | 弱加密、明文存储 |
| A03 | **注入攻击** | SQL 注入、命令注入、LDAP 注入 |
| A04 | **不安全设计** | 架构层面的安全缺陷 |
| A05 | **安全配置错误** | 默认口令、对外开放的管理端 |
| A06 | **易受攻击的组件** | 第三方库漏洞 |
| A07 | **认证和身份验证失败** | 弱口令、会话泄露 |
| A08 | **数据完整性失效** | 不验证数据来源 |
| A09 | **安全日志和监控不足** | 无法发现攻击 |
| A10 | **服务端请求伪造（SSRF）** | 利用服务端发起内网请求 |

---

## 重点攻防

### SQL 注入
```sql
-- 攻击：输入 ' OR '1'='1
SELECT * FROM users WHERE name = 'admin' OR '1'='1'
```
**防御：** 参数化查询、ORM、输入过滤

### XSS（跨站脚本）
```html
<script>alert('xss')</script>
```
**类型：** 反射型、存储型、DOM 型
**防御：** 输出编码、CSP（内容安全策略）

### CSRF（跨站请求伪造）
- 利用用户的已登录状态，诱导其执行非本意的操作
- **防御：** CSRF Token、SameSite Cookie

### SSRF（服务端请求伪造）
- 让服务器向内网发起请求，探测内部服务
- **防御：** URL 白名单、禁止内网 IP 请求

---

## 实践建议

1. **输入永远不可信** — 所有用户输入都需要校验和清洗
2. **最小信息暴露** — 错误提示不要说具体原因（别说"密码错误"，只说"登录失败"）
3. **默认安全配置** — 安装框架库就用安全默认配置
4. **依赖库管理** — 使用 SCA（软件成分分析）扫描漏洞
5. **安全头配置** — CSP、X-Frame-Options、HSTS

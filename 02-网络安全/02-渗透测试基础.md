# 渗透测试基础

> 用攻击者的思维来检验防御的有效性。

---

## 渗透测试阶段

```
信息收集 ──→ 漏洞分析 ──→ 漏洞利用 ──→ 权限维持 ──→ 报告
```

### 1. 信息收集（Reconnaissance）
- 被动：OSINT、Whois、子域名枚举
- 主动：端口扫描（nmap）、服务识别
- 工具：nmap、masscan、shodan、theHarvester

### 2. 漏洞分析（Vulnerability Assessment）
- 自动扫描：Nessus、OpenVAS
- 手动验证：确认漏洞是否真实可利用

### 3. 漏洞利用（Exploitation）
- Metasploit、Burp Suite
- SQL 注入、XSS、命令执行
- 提权（Privilege Escalation）

### 4. 权限维持（Persistence）
- 反弹 Shell、后门安装
- 横向移动

### 5. 报告（Reporting）
- 漏洞详情和风险评级
- 修复建议
- 复测确认

---

## ⚠️ 合法性问题

- **必须有书面授权**才能测试
- 未经授权渗透测试 = 违法（违反《网络安全法》）
- 使用漏洞扫描工具也要注意：扫描即攻击

---

## 学习建议

1. **从小靶场开始** — DVWA、Vulnhub、HackTheBox
2. **先懂防御再学攻击** — 知道 WAF 怎么防才知道怎么绕过
3. **持续保持合规意识** — 只在授权范围内操作

# WAF 与抗 DDoS

> Web 应用安全的两道重要防线。

---

## WAF（Web 应用防火墙）

### 部署位置
```
用户 ─→ WAF ─→ Web 应用服务器 ─→ API ─→ 数据库
```

### 检测方式
| 方式 | 原理 | 优缺点 |
|------|------|--------|
| **规则匹配** | 基于签名库匹配攻击载荷 | 准确但依赖更新 |
| **语义分析** | 理解 HTTP 语义和行为 | 能防变种攻击 |
| **机器学习** | 基于基线行为异常检测 | 低误报但需要训练 |

### 云 WAF vs 硬件 WAF
| 对比 | 云 WAF | 硬件 WAF |
|------|--------|----------|
| 部署 | DNS/CNAME 接入 | 串接在网络链路 |
| 弹性 | 自动伸缩 | 受硬件性能限制 |
| TLS 卸载 | 支持 | 视型号而定 |
| 成本 | 按需付费 | 一次性投入 |
| 维护 | 厂商负责 | 自行运维 |

---

## DDoS（分布式拒绝服务）

### 攻击类型

| 层 | 类型 | 攻击方法 |
|----|------|---------|
| **L3/L4** | 容量耗尽型 | SYN Flood、UDP Flood、ICMP Flood |
| **L7** | 应用层攻击 | HTTP Flood、慢速攻击 |
| **反射放大** | 中间人利用 | NTP/DNS/Memcached 放大 |

### 防御分层

```
L3/L4 DDoS → 云清洗中心（阿里云高防/AWS Shield）
  ↓
L7 DDoS → WAF + CDN（过滤恶意请求）
  ↓
应用层限流 → Rate Limiting + 熔断
  ↓
业务层 → 风控系统
```

---

## ⚠️ 操作陷阱

1. **WAF 规则太严** — 可能误拦截正常业务请求，需充分测试
2. **DDoS 清洗不及时** — 从"检测到流量异常"到"引流到清洗中心"的响应时间决定了业务受损程度
3. **忽略了 L7 DDoS** — 100Mbps 的 HTTP Flood 比 100Gbps 的 SYN Flood 更难防御
4. **仅靠 WAF 防不住业务逻辑漏洞** — 比如刷单、优惠券套利

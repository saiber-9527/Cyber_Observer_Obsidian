# AI Agent 基础

> 从"我问一句你答一句"到"你帮我干完一件事"。

---

## 什么是 Agent

```
普通 LLM： 你问 → 答完 → 结束
Agent：    你给目标 → 模型自己规划步骤 → 调用工具执行 → 完成
```

**Agent = 大模型 + 工具 + 记忆 + 自主规划**

---

## Agent 是如何工作的

```
用户："帮我查一下这个 IP 是不是有问题"

Agent 内部：
  ① 思考："我需要查 IP 信誉，然后可能还要查关联域名"
  ② 行动：调用威胁情报 API（function calling）
  ③ 观察：API 返回该 IP 有恶意活动记录
  ④ 思考："需要再查一下具体的恶意类型"
  ⑤ 行动：去病毒库查详细信息
  ⑥ 输出："这个 IP 被标记为 C2 服务器...建议..."
```

**这种"思考→行动→观察→再思考"的模式叫做 ReAct。**

## Function Calling（让 Agent 连接外部工具）

```python
# 定义工具
tools = [
    {
        "name": "query_threat_intel",
        "description": "查IP信誉",
        "parameters": { "ip": "string" }
    },
    {
        "name": "send_alert",
        "description": "发送告警到飞书",
        "parameters": { "message": "string" }
    }
]

# Agent 收到请求后，自己决定要不要调用工具
# 它不会直接执行代码，而是返回要调用的函数名和参数
# 由外层系统去执行
```

## 安全 Agent 的典型场景

| 场景 | 说明 |
|------|------|
| **自动威胁研判** | Agent 自动查 IP、查域名、查样本，生成研判结论 |
| **事件自动响应** | 发现告警 → 隔离主机 → 通知负责人 |
| **漏洞扫描助手** | Agent 帮你分析扫描结果，排优先级 |
| **合规检查工具** | Agent 读配置，判断是否符合等保要求 |

---

## ⚠️ Agent 安全风险（重要）

1. **工具误调用** — 如果 Agent 决定调用"删除数据库"工具——后果很严重
2. **提示注入传播** — 用户恶意输入 → Agent 去执行危险操作
3. **无限循环** — Agent 陷入"继续思考→继续行动→继续思考"的死循环
4. **权限管控** — Agent 的 API Key 不能比人类操作员的权限更大

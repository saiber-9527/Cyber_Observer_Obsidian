# LLM 与大模型基础

> 大规模语言模型—今天 AI 应用的核心引擎。

---

## 训练流程三个阶段

```
第一阶段：预训练（Pre-training）
  海量互联网文本 → 学习语言规律和世界知识
  成本：数千万美元级别

第二阶段：监督微调（SFT）
  高质量的指令-回答对 → 学会遵循指令
  成本：数十万美元级别

第三阶段：RLHF（人类反馈强化学习）
  人类偏好数据 → 对齐人类价值观
  成本：十万美元级别
```

## 关键概念

| 概念 | 说明 |
|------|------|
| **上下文窗口** | 模型一次能看到的 token 数量 |
| **Token** | 文本的基本单位（一个字 ≈ 1-2 tokens，一个英文词 ≈ 1 token） |
| **Temperature** | 控制输出的随机性（低=确定、高=创意） |
| **Top-p (nucleus sampling)** | 控制候选词的范围（只从累计概率 p 内的词采样） |
| **System Prompt** | 给模型的系统级别指令（设定角色和行为边界） |
| **Hallucination（幻觉）** | 模型生成看似合理但实际错误的内容 |

## 推理优化技术

| 技术 | 说明 |
|------|------|
| **KV Cache** | 缓存已计算过的注意力键值，避免重复计算 |
| **量化** | 减少权重的精度（FP16 → INT8/INT4），显著降低显存 |
| **连续批处理** | 多个请求合并为一个批次处理，提高吞吐 |
| **推测解码** | 用小模型生成候选，大模型一次验证 |
| **FlashAttention** | 高效计算注意力，减少显存读写 |

## 微调方法

| 方法 | 说明 | 需要算力 |
|------|------|---------|
| **全参微调** | 更新模型全部参数 | 很高 |
| **LoRA/QLoRA** | 冻结原参数，插入少量可训练参数 | 低（单卡可跑） |
| **Prompt Tuning** | 只训练 prompt 的嵌入向量 | 极低 |

---

## ⚠️ 安全相关的关键事实

1. **模型会学到训练数据中的偏见和有害内容** — 需要安全对齐
2. **上下文窗口越大，越容易被利用** — 长文本攻击面更大
3. **微调可能覆盖安全对齐** — 即使之前已经做了安全对齐
4. **非开源模型的 API 调用** — 数据会发送到厂商服务器（数据隐私问题）

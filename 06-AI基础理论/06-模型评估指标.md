# 模型评估指标

> 模型好不好，不能靠感觉说——要有数字。

---

## 分类任务

### 混淆矩阵

```
               实际正类    实际负类
预测正类         TP         FP
预测负类         FN         TN
```

### 核心指标

| 指标 | 公式 | 含义 | 适用场景 |
|------|------|------|---------|
| **准确率 (Accuracy)** | (TP+TN)/(TP+FP+FN+TN) | 整体正确率 | 正负样本均衡 |
| **精确率 (Precision)** | TP/(TP+FP) | 预测为正类的结果中，真的正类占比 | 垃圾邮件过滤（宁可漏不能误） |
| **召回率 (Recall)** | TP/(TP+FN) | 在所有正类中，模型找出了多少 | 安全检测（宁可误不能漏） |
| **F1 分数** | 2×P×R/(P+R) | 精确率和召回率的调和平均 | 综合评估 |
| **特异度 (Specificity)** | TN/(TN+FP) | 负类被正确识别的比例 | 医疗诊断 |

### 安全领域的特殊考量

安全场景中，**召回率通常比精确率更重要**：
- 漏报一个攻击（FN）= 可能造成实际损失
- 误报一个攻击（FP）= 分析师去核实一下，可以接受

但误报太多也不行——告警疲劳会让真正的问题被忽略。

### ROC 曲线与 AUC

- **ROC 曲线**：不同阈值下的 TPR（召回率）vs FPR（1-特异度）
- **AUC**：ROC 曲线下的面积
- **AUC = 1** → 完美分类器
- **AUC = 0.5** → 随机猜测

## 回归任务

| 指标 | 说明 |
|------|------|
| **MSE** (均方误差) | 预测值与真实值差值的平方平均 |
| **MAE** (平均绝对误差) | 预测值与真实值差值的绝对值的平均 |
| **RMSE** | MSE 开根号，和原始数据同一量级 |
| **R²** | 模型解释了多少方差（越接近 1 越好） |

---

## LLM 评估

| 指标 | 说明 |
|------|------|
| **Perplexity** | 模型对下一个 token 预测的困惑程度（越低越好） |
| **Benchmark 分数** | MMLU、HumanEval、GSM8K 等标准化测试 |
| **BLEU / ROUGE** | 生成文本与参考文本的相似度 |
| **人工评估** | 人类标注者评分（最可靠但最贵） |
| **LLM-as-a-Judge** | 用另一个 LLM 来打分 |

## 过拟合 vs 欠拟合

```
训练集准确率 ↑  训练集准确率 ↓
测试集准确率 ↓  测试集准确率 ↓
────────────────────────────────
   过拟合          欠拟合

过拟合：模型背下了训练数据，不会泛化
欠拟合：模型连训练数据都没学好
```

**解决方法：**
| 问题 | 解法 |
|------|------|
| 过拟合 | 减少模型复杂度、增加数据量、正则化 (L1/L2)、Dropout |
| 欠拟合 | 增加模型复杂度、减少正则化、更长的训练 |

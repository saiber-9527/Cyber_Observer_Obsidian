# 预训练技术详解

> LLM 能力的基石——海量数据 + 大规模算力 + 自监督学习。

---

## 预训练的核心思路

```
传统 ML：你给模型标注好的数据（"这张图是猫"）
预训练：不需要标注，让模型从数据中"自己学规律"
        （"看了一万亿个词，自然学会了语法和知识"）

所以叫"自监督学习"——数据既是输入又是标签。
```

---

## 预训练任务

### 1. 因果语言模型（Causal LM）
GPT 系使用的方式：

```
输入： "今天天气真"
目标： 预测下一个词"好"

过程：
  "今" → 模型预测下一个词 → 应该是"天"
  "今天" → 模型预测下一个词 → 应该是"天"
  "今天天" → 模型预测下一个词 → 应该是"气"
  "今天天气" → 模型预测下一个词 → 应该是"真"
  "今天天气真" → 模型预测下一个词 → 应该是"好"
```

**特点：** 只能看前面的词（单向注意力），适合文本生成

### 2. 遮蔽语言模型（Masked LM）
BERT 系使用的方式：

```
输入： "今天天气[MASK]好"
目标： 预测被遮住的词"真"

特点： 可以看上下文（双向注意力），适合文本理解
```

### 3. 其他预训练任务
| 任务 | 说明 | 适用模型 |
|------|------|---------|
| 下一句预测 | 判断两句话是否连续 | BERT |
| 文本填充 | 预测被遮住的连续片段 | T5 |
| 排序预测 | 预测打乱的句子顺序 | ALBERT |
| 对比学习 | 让相似文本的表示靠近 | SimCSE |

## 训练数据

### 数据组成

```
互联网爬取（CommonCrawl）  ── 约 60%
  （质量参差不齐，需要大量清洗）
  
精选数据：
  ├── Wikipedia         ── 10% （高质量百科）
  ├── 书籍/论文          ── 10% （深度长文本）
  ├── 代码（GitHub）     ── 10% （代码理解能力）
  ├── 社交媒体           ── 5%  （对话风格）
  └── 专有数据           ── 5%  （特定领域）
```

### 数据清洗

```python
# 预训练数据清洗流水线（简化版）
def clean_pretraining_data(text):
    # 1. 去重（MinHash/LSH 去近似重复）
    if is_near_duplicate(text):
        return None
    
    # 2. 质量过滤
    if len(text) < 200:           # 太短不要
        return None
    if perplexity_score(text) > 5000:  # 质量太差不要（用已有模型评估）
        return None
    if contains_pii(text):         # 含个人信息
        text = redact_pii(text)
    
    # 3. 格式标准化
    text = normalize_unicode(text)
    text = dedent_code_blocks(text)
    
    return text
```

## 分布式训练技术

```
数据并行（Data Parallel）：
  每个 GPU 有一份完整模型副本
  每个 GPU 处理不同的数据批次
  每步同步梯度 → 更新所有 GPU

模型并行（Model Parallel）：
  模型太大装不进单卡
  将不同层放在不同 GPU 上
  数据在 GPU 间串行传递

张量并行（Tensor Parallel）：
  将矩阵运算拆分到多个 GPU
  这一步在同一个层内做切分

流水线并行（Pipeline Parallel）：
  将模型切分成多个阶段
  每个阶段在单独的 GPU 上
  数据像流水线一样在各阶段流动
```

## 预训练的挑战

| 挑战 | 说明 |
|------|------|
| **算力成本** | 训练一次 70B 模型 = 几千万美元电费 |
| **数据质量** | 互联网数据质量参差不齐，清洗非常困难 |
| **训练不稳定** | 大模型训练容易"崩"（loss 发散），需要大量调参经验 |
| **硬件故障** | 数千 GPU 运行数月，总会有硬件故障，需要 checkpoint 恢复 |
| **环境影响** | 训练一次大模型的碳排放 ≈ 500 辆车一年的排放量 |
| **法律风险** | 训练数据是否有版权、是否含敏感内容 |

# RLHF（基于人类反馈的强化学习）深度解析

> 让 AI 学会"什么该说，什么不该说"——ChatGPT 背后的关键对齐技术。

---

## 为什么需要 RLHF

```
预训练完成后的模型：知道很多知识，但不听用户指挥

用户：帮我写一封邮件
模型：我今天心情不错，天气也很好，你想听个笑话吗？
          ↑
   模型完全不知道"回答要符合用户预期"
```

**SFT（监督微调）可以教它格式，但教不了"你觉得哪个回答更好"。**
RLHF 的解决方案：让人类告诉模型"我喜欢这个回答胜过那个"。

---

## RLHF 三阶段

### 第一阶段：SFT（监督微调）

```
数据：人工标注的"问题-期望回答"对
目标：模型学会遵循指令的格式和风格
成本：标注约 1-10 万条数据

作用：让模型从"一个会说人话的语料库"变成"一个会回答问题的助手"
局限：模型学的是"标准答案"，不是"什么答案更好"
```

### 第二阶段：奖励模型（Reward Model）训练

```
数据：人类标注者对同一问题的多个回答进行排序
例如：
  问题："如何做好密码安全？"
  回答A："用复杂密码，包含大小写数字特殊字符"          ← 标注者排第3
  回答B："密码安全的三要素：长度、复杂度、唯一性..."   ← 标注者排第1
  回答C："密码啊，随便设一个就行了"                   ← 标注者排最后
  
训练：让奖励模型学会"好回答"和"坏回答"的区别
成本：标注约 10-100 万条偏好数据
```

### 第三阶段：强化学习（PPO）

```
训练过程：

1. 当前策略（语言模型）生成回答
2. 奖励模型对该回答打分
3. PPO 算法更新语言模型的参数
4. 重复 1-3

关键：PPO 强制新策略不要偏离旧策略太远
      （防止模型为了刷分而学了一些奇怪的模式）
```

## PPO 算法的直观理解

```python
# PPO 的简化理解（实际复杂得多）

# 语言模型生成回答 A
response = model.generate(prompt)

# 奖励模型打分
reward = reward_model(prompt, response)  # 得分 0.85

# 计算新旧策略的差异比例
ratio = new_policy_prob(response) / old_policy_prob(response)

# 如果 ratio 太大（新策略变化太大），clip 掉
clipped_ratio = clamp(ratio, 0.8, 1.2)

# 损失 = -min(ratio * reward, clipped_ratio * reward)
# 最大化奖励，但限制策略不要飘太远
```

## RLHF 的挑战

| 挑战 | 说明 |
|------|------|
| **标注成本高** | 高质量的人类偏好标注需要专业的标注员（$10-50/小时） |
| **奖励模型偏差** | 奖励模型本身也有偏见和错误 |
| **奖励欺骗** | 模型可能学出"刷分"策略而非真正变好 |
| **过度对齐** | 模型过于讨好人类，即使人类要求的是错误答案 |
| **对齐扩散** | 对齐过的模型可以微调破坏对齐 |

---

## RLHF 的替代方案

| 方案 | 说明 | 代表 |
|------|------|------|
| **RLHF** | 人类标注 + 奖励模型 + PPO | ChatGPT、Claude |
| **Constitutional AI** | 不依赖人类标注，让模型按"宪法"自我修正 | Claude 2+ |
| **DPO** | 直接偏好优化，不需要奖励模型 | Zephyr、Llama 3 |
| **KTO** | 仅用"好"和"不好"的标签，不需要排序 | — |
| **RLAIF** | 用另一个 LLM 替代人类标注 | Anthropic |

## 安全对齐的局限

```
RLHF 做了安全对齐的模型 = 一个知道"不该做什么"的模型

但：
  └── 可以 jailbreak（越狱）：DAN prompt、角色扮演
  └── 可以通过微调解除对齐：在少量不安全数据上微调
  └── 可以通过 fine-tuning 绕过：LoRA 微调几美元就能解除对齐
  └── 提示注入：系统 prompt 可以被覆盖

所以：
  安全对齐 ≠ 绝对安全
  安全对齐 + 输入过滤 + 输出过滤 + 监控 = 相对安全
```

## ⚠️ 关键认知

1. **RLHF 让模型更有用，但也更狡猾** — 它学会了讨好评分者
2. **奖励模型本身就是安全漏洞** — 如果奖励模型有偏见，对齐也会学偏
3. **对齐是猫鼠游戏** — 攻击者不断发明新的绕过方式，对齐需要持续升级
4. **开源模型没有对齐** — 任何人都可以下载基座模型，从头微调出"无安全限制"的版本

# 数据集与数据标注

> 没有好数据，再好的模型也白搭。数据质量决定模型上限。

---

## 数据集的三个角色

```
训练集 (Training Set)  — 用于训练模型参数
验证集 (Validation Set) — 用于调参数和选择模型
测试集 (Test Set)       — 用于最终评估（训练期间绝不接触）
```

比例：通常 80%/10%/10% 或 70%/15%/15%

---

## 常见误导（需要警惕）

```
问题：为什么测试集准确率 99%，上线后完全不能用？

可能的原因：
1. 训练集和测试集来自同一分布，但生产数据分布不同
   （如：训练用的都是白天数据，模型晚上表现很差）
   
2. 数据泄露（Data Leakage）：
   训练集中包含了测试集的信息
   （如：用全部数据做归一化后再分训练/测试集）

3. 标签泄露（Label Leakage）：
   特征中包含了未来的信息
   （如：用"是否已报警"来预测"是否有攻击"——已经报警了当然有攻击）
```

## 数据标注

### 标注类型

| 类型 | 说明 | 典型成本 |
|------|------|---------|
| **分类标注** | 给数据打标签 | 低 |
| **标注框 (Bounding Box)** | 在图像中标出目标位置 | 中 |
| **语义分割** | 像素级标注 | 高 |
| **文本标注** | 实体识别、关系标注 | 中-高 |
| **偏好排序** | RLHF 中使用的人类偏好比较 | 高（需要专业标注员） |

### 安全数据标注的难点

```
邮件安全分类：
  "这封邮件是钓鱼邮件吗？"
  
  难点：
  - 钓鱼邮件不断变种，旧的标签不一定适用
  - 正常的营销邮件和钓鱼邮件之间界限模糊
  - 标注员需要安全专业知识才能准确判断
  - 漏标一条钓鱼邮件 = 潜在安全事故
  
  建议做法：
  - 多人标注 + 一致性校验
  - 定期复盘和修正标签
  - 建立难样本库（标注员分歧大的样本单独处理）
```

## 数据不平衡问题

```
场景：网络入侵检测数据集
  正常流量：99.9%
  入侵流量：0.1%
  
问题：模型全预测"正常"就有 99.9% 准确率
     但这模型没有任何检测能力！
```

### 处理方法

| 方法 | 说明 | 适用 |
|------|------|------|
| **下采样** | 减少多数类样本 | 数据量大时 |
| **上采样** | 复制少数类样本 | 数据量小时 |
| **SMOTE** | 合成新的少数类样本 | 特征空间有效时 |
| **加权损失** | 给少数类样本更高权重 | 通用方法 |
| **Focal Loss** | 关注难分样本 | 严重不平衡 |
| **异常检测算法** | 把问题转为异常检测 | 极度不平衡 |

## ⚠️ 常见数据问题

| 问题 | 影响 | 排查方法 |
|------|------|---------|
| 标注错误 | 模型学习错误模式 | 随机抽检标注质量 |
| 重复数据 | 高估模型性能 | 去重检查 |
| 数据偏差 | 模型预测偏差 | 分析数据分布 |
| 时间不一致 | 知识过期 | 按时间拆分验证 |
| 特征缺失 | 模型性能下降 | 统计缺失率 |

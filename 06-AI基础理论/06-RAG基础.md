# RAG（检索增强生成）基础

> 让 LLM 在回答前"查一下资料"，而不是全靠记忆。

---

## 为什么需要 RAG

LLM 的两个硬伤：
1. **知识截止** — 不知道训练之后的新事
2. **幻觉** — 不知道的会胡编

RAG 的解决思路：别让模型猜——先把相关资料找出来喂给它。

---

## 核心流程

```
你问："公司今年的数据安全要求是什么？"
              │
   ① 把你的问题变成向量（Embedding）
              │
   ② 去向量数据库里搜最相关的文档片段
              │
   ③ 把搜到的资料 + 你的问题一起发给 LLM
              │
   ④ LLM 基于资料回答，附上引用来源
```

---

## 简单理解

```
RAG = 谷歌搜索 + ChatGPT
     ↖         ↗
   先搜一下   再回答
```

模型本身的训练数据是"通识教育"，RAG 加上的是"开卷考试材料"。

---

## 适合的场景

| 场景 | 为什么用 RAG |
|------|-------------|
| 企业内部知识库问答 | 知识更新快，不适合频繁微调 |
| 实时信息查询 | 最新新闻、产品文档 |
| 法律/医疗等需要准确引用的场景 | 回答要有出处 |
| 多文档问答 | 在多个文档中找答案 |

## 不适合的场景

| 场景 | 原因 |
|------|------|
| 需要模型"学会"新技能 | 应该用微调 |
| 统一风格/行为调整 | 微调更适合 |
| 超低延迟场景 | RAG 多一次检索调用 |

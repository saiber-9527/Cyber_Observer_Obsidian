# 容器镜像安全

> 镜像不安全，跑起来的容器天生带毒。

---

## 镜像安全风险树

```
Dockerfile ──→ 基础镜像 ──→ 构建 ──→ 镜像仓库 ──→ 运行
   │              │          │          │          │
   层泄露         漏洞       构建      镜像投毒     运行时逃逸
   密钥嵌入      恶意组件   缓存污染   版本混淆
   高危命令
```

---

## Dockerfile 安全最佳实践

### ❌ 不安全的 Dockerfile

```dockerfile
FROM ubuntu:latest           # 👎 "latest" 不可控
RUN apt-get update           # 👎 未清理缓存
RUN apt-get install -y wget curl netcat vim  # 👎 装了很多不必要的包
COPY . /app                  # 👎 把整个目录（可能含 .env/密钥）复制进去
COPY private.key /app/       # 👎 把密钥打包进镜像
RUN pip install -r requirements.txt  # 👎 未扫描依赖漏洞
CMD ["python", "app.py"]     # 👎 以 root 运行
```

### ✅ 安全的 Dockerfile

```dockerfile
# 选具体的版本号（不用 latest）
FROM python:3.11-slim-bookworm

# 设置标签
LABEL maintainer="team@example.com" \
      version="1.0.0"

# 使用非 root 用户
RUN groupadd -r appgroup && useradd -r -g appgroup appuser

# 设置工作目录
WORKDIR /app

# 只复制需要的内容（利用缓存）
COPY requirements.txt .

# 安装依赖（清理缓存减少镜像大小）
RUN pip install --no-cache-dir -r requirements.txt && \
    rm -rf /root/.cache/pip

COPY src/ ./src/

# 设置权限
RUN chown -R appuser:appgroup /app

# 切换到非 root 用户
USER appuser

CMD ["python", "src/app.py"]
```

## 多阶段构建（减少攻击面）

```dockerfile
# 第一阶段：构建
FROM golang:1.21 AS builder
WORKDIR /build
COPY . .
RUN CGO_ENABLED=0 go build -o app .

# 第二阶段：运行（使用最小基础镜像）
FROM alpine:3.19
RUN apk add --no-cache ca-certificates tzdata
COPY --from=builder /build/app /app
USER nobody
CMD ["/app"]
```

**好处：** 最终镜像只有运行所需的最小文件（没有编译器、依赖源码）

## 镜像扫描

```bash
# Trivy 扫描（最常用的开源扫描器）
# 扫描本地镜像
trivy image nginx:1.25-alpine

# 扫描 Dockerfile 中的基础镜像
trivy filesystem --severity CRITICAL,HIGH .

# 扫描容器运行时
trivy container my-running-container

# 输出为 JSON 用于集成
trivy image --format json -o report.json nginx:latest

# 在 CI/CD 中设置扫描失败条件
trivy image --exit-code 1 --severity CRITICAL nginx:latest
```

## 镜像签名与验证

```bash
# 使用 Cosign 签名镜像（Sigstore 项目）

# 生成密钥对
cosign generate-key-pair

# 签名镜像
cosign sign --key cosign.key registry.example.com/myapp:v1.0

# 验证签名
cosign verify --key cosign.pub registry.example.com/myapp:v1.0

# 在 K8s 中强制验证签名
# 配置准入控制器只允许已签名的镜像运行
```

## 镜像安全清单

```
□ 基础镜像使用特定版本号（不用 latest）
□ 使用最小基础镜像（alpine/distroless）
□ 多阶段构建分离编译环境和运行环境
□ 容器不以 root 运行（USER 指令）
□ Dockerfile 中不包含密钥/密码
□ 定期扫描镜像（Trivy/Snyk）
□ CI/CD 中设置扫描失败门槛（如 Critical 必须解决）
□ 使用签名验证镜像完整性
□ 定期重建镜像（基础镜像的补丁会更新）
□ 避免在镜像中安装不必要的软件包/调试工具
```

---

## ⚠️ 高优先级陷阱

1. **"latest" 标签不可控** — 今天 build 和三个月后 build 可能得到完全不同的环境
2. **缓存中毒** — CI 中的 Docker 缓存可能包含旧的漏洞
3. **镜像层暴露敏感信息** — Docker 每一层都是独立的文件系统，删除文件只是在上面覆盖一层
4. **只有 build 时有安全扫描不够** — 基础镜像更新了，你的镜像不会自动更新

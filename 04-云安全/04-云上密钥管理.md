# 云上密钥管理（KMS）

> 密钥管理是最重要也最容易被忽视的安全环节。管理好密钥 = 管好数据的最后一道防线。

---

## 为什么需要 KMS

```
本地管理密钥的问题：
1. 密钥写死在代码里 → 代码仓库泄露 = 全部沦陷
2. 密钥存在服务器文件里 → 服务器被入侵 = 密钥泄露
3. 密钥轮换困难 → 手动更新容易遗漏或出错
4. 审计缺失 → 不知道谁用了密钥

KMS 的解决方案：
1. 密钥集中管理，不离开 KMS 硬件
2. API 调用使用密钥而非密钥本身
3. 自动轮换
4. 完整的操作审计
```

---

## 信封加密

KMS 的核心加密方式：

```
                  主密钥（CMK）
                      │
              ┌───────┴───────┐
              ↓               ↓
        数据密钥（DEK）     加密 DEK
              │               │
        加密数据           存到数据库
```

**流程：**
1. 调用 KMS 生成一个数据密钥（DEK）
2. KMS 用主密钥（CMK）加密 DEK，返回密文版 DEK
3. 用 DEK 明文加密数据
4. 丢弃 DEK 明文，只保存加密后的 DEK 和加密后的数据
5. 解密时：用 CMK 解密 DEK 密文 → 得到 DEK 明文 → 解密数据

**为什么这样设计：** CMK 始终不离开 KMS 服务器，密钥泄漏风险降至最低。

---

## KMS 核心能力

| 能力 | 说明 |
|------|------|
| 对称密钥 | 加密/解密大量数据 |
| 非对称密钥 | 签名/验签，证书签发 |
| 密钥轮换 | 自动定期更换 CMK |
| 密钥删除 | 安全删除（有等待期，可撤销） |
| 跨区域 | 不同 Region 使用不同 CMK |
| 访问控制 | 集成 IAM，细粒度控制谁能用密钥 |

## KMS + 云服务集成

```yaml
OSS 服务端加密: OSS 使用 KMS 自动加密上传的文件
RDS 透明数据加密: 数据库使用 KMS 自动加密存储
ECS 云盘加密: 系统盘/数据盘使用 KMS 加密
K8s Secret 加密: 用 KMS 加密 etcd 中的 Secret
SLS 日志加密: 日志服务使用 KMS 加密写入
```

## 密钥管理最佳实践

```
✅ 必须做的：
  ├── 主密钥开启自动轮换（1 年一次）
  ├── 最小权限授予 KMS 调用权限
  ├── 启用 KMS 操作审计（谁在什么时候调用了哪个密钥）
  ├── 定期审查密钥使用情况
  └── 密钥删除前设置等待期（默认 7-30 天）

❌ 不要做的：
  ├── 把 AccessKey/SecretKey 硬编码在代码里
  ├── 使用同一个密钥加密所有数据
  ├── 将密钥明文存入配置文件
  └── 将 KMS 密钥共享给第三方应用
```

## 密钥生命周期

```
创建 CMK → 启用 → 使用（加密/解密）→ 禁用（临时）→ 删除
   │                                         │
   │←────────── 自动轮换 ─────────────────→│
                   (每年)
```

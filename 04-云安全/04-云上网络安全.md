# 云上网络安全

> 云上的网络和传统物理网络不一样——软件定义、API 驱动、灵活但也更容易配错。

---

## VPC（虚拟私有网络）

### 核心概念

| 概念 | 说明 |
|------|------|
| **VPC** | 云上的隔离网络空间，相当于一个逻辑机房 |
| **交换机/子网** | VPC 内的 IP 网段划分 |
| **路由表** | 控制 VPC 内和外部的流量方向 |
| **NAT 网关** | 让私有网络内的资源访问互联网 |
| **VPN/专线** | 连接云上和本地数据中心 |
| **EIP** | 弹性公网 IP，可绑定到不同的云资源 |

### 安全组 vs 网络 ACL

| 对比 | 安全组（SG） | 网络 ACL（NACL） |
|------|------------|------------------|
| 层级 | 实例级别（第一道防线） | 子网级别（第二道防线） |
| 状态 | **有状态**（允许出=自动允许回应） | **无状态**（需双向规则） |
| 规则 | 仅允许规则 | 允许+拒绝规则 |
| 评估 | 所有规则合并评估 | 按编号顺序（最先匹配优先） |
| 适用 | 精细控制单个实例 | 子网级别的全局规则 |

### 安全组最佳实践

```bash
# 最小开放原则
# 只放行业务需要的端口

# Web 服务器安全组
Inbound:
  HTTP (80)  — 0.0.0.0/0
  HTTPS (443) — 0.0.0.0/0
  SSH (22)   — 管理网段（非 0.0.0.0/0！）
  
# 数据库安全组
Inbound:
  MySQL (3306) — 仅允许应用服务器安全组
  
# 禁止公网直接连接数据库！
```

## 常见网络配置错误

| 错误 | 后果 |
|------|------|
| SSH/3389 对所有 IP 开放 | 服务器沦为爆破目标 |
| RDS/Redis 公网可访问 | 数据库暴露 |
| 安全组规则过多 | 审计困难，容易出错 |
| 没设 NAT 网关 | 私有网络的服务器无法访问外网（如 yum 更新） |
| VPC 对等连接不对 | 跨账号/跨区域网络不通 |

## 流量镜像与日志

- **VPC Flow Logs / 流日志** — 记录网络流量元数据，用于排查和安全分析
- **流量镜像** — 将特定端口的流量复制一份到分析工具

```bash
# 流日志配置要点
# 1. 开启 VPC/Flow Logs
# 2. 写入日志服务（SLS/CloudWatch）
# 3. 设置告警规则（如：检测到外连异常 IP）
```

## 云网络架构安全建议

```
1. 不同业务用不同 VPC
2. 灰度环境和生产环境网络隔离
3. 数据库类服务只允许内网访问
4. 管理端使用堡垒机（非直接 SSH）
5. API 端点使用 API 网关统一管理
6. 公网入口统一经过 WAF + SLB
```

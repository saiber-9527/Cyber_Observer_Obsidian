# 对抗攻击与 AI 红队

> 主动测试模型的安全边界。

---

## 对抗攻击（Adversarial Attacks）

### 攻击面

| 攻击阶段 | 类型 | 说明 |
|---------|------|------|
| **训练前** | 数据投毒 | 在训练数据中植入恶意样本 |
| **训练中** | 后门植入 | 特定触发器激活恶意行为 |
| **推理时** | 对抗样本 | 在输入上做微扰，导致错误分类 |
| **推理时** | 模型窃取 | 通过 API 调用重建近似模型 |

### 对抗样本示例（图像）

```
原始图片：🐱 → 模型正确识别为"猫"
添加噪声后：🐱 +🫥 → 人眼看还是猫，但模型识别为"鸵鸟"
```

### 模型窃取

- 通过大量 API 调用和输出，反向训练一个功能相近的模型
- 成本约为原模型训练成本的 1/100 到 1/1000
- 防御：限制 API 调用频率、输出水印化

## AI 红队测试

### 测试目标
1. **安全对齐** — 模型是否会生成有害内容
2. **鲁棒性** — 面对对抗样本的稳定性
3. **公平性** — 是否存在性别/种族偏见
4. **可靠性** — 幻觉率和事实准确性

### 测试方法论

| 方法 | 说明 |
|------|------|
| **手工红队** | 安全专家手动设计攻击 |
| **自动化红队** | 用另一个 LLM 自动生成攻击 prompt |
| **强化学习红队** | RL 训练攻击 Agent |
| **LLM 评分器** | 用 LLM 评估另一个 LLM 的输出安全性 |

### 评估基准

| 基准 | 说明 |
|------|------|
| **HARMBENCH** | 有害内容生成测试 |
| **TruthfulQA** | 事实准确性测试 |
| **MMLU** | 综合知识评估 |
| **BOLD** | 偏见检测 |
| **Red Team API** | 微软的自动化红队工具 |

---

## ⚠️ 实践注意

1. **AI 红队测试需要伦理审查** — 测试用的攻击 prompt 本身可能造成模型学到更多攻击方式
2. **红队测试是持续的** — 不是一次性的发布前检查，模型可能会被微调后"失忆"
3. **自动化红队不能完全替代人工** — 人类专家能发现自动化工具发现不了的攻击模式

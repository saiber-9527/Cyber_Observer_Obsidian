# AI 内容安全

> 模型输出什么，比模型能做什么更重要。

---

## 内容安全风险

| 风险类型 | 示例 | 影响 |
|---------|------|------|
| **有害内容** | 暴力、仇恨言论、色情 | 违反法律法规 |
| **错误信息** | 医疗建议、法律建议 | 误导用户，造成实际伤害 |
| **偏见歧视** | 种族/性别/地域偏见 | 公平性问题，声誉损害 |
| **商业秘密** | 模型输出了训练数据中的机密 | 侵权风险 |
| **幻觉** | 编造事实、虚假引用 | 信任度下降 |
| **版权侵权** | 逐字输出受版权保护的文本 | 法律风险 |

---

## 防御措施

### 输入层
| 措施 | 说明 |
|------|------|
| **关键词过滤** | 拦截已知有害词 |
| **语义检测** | ML 分类器判断输入意图 |
| **rate limit** | 限制频率，防批量攻击 |

### 模型层
| 措施 | 说明 |
|------|------|
| **安全对齐** | RLHF / 宪法 AI |
| **安全微调** | 在安全相关数据上额外训练 |
| **系统 prompt 加固** | 固定角色和边界 |

### 输出层
| 措施 | 说明 |
|------|------|
| **内容审核 API** | 调用外部审查服务 |
| **敏感词后处理** | 正则匹配拦截 |
| **质量检查** | 检测连贯性、事实性 |
| **用户反馈** | "这个回答有问题"按钮 |

---

## 内容审核流水线

```
用户输入 → 输入过滤 → 模型推理 → 输出过滤 → 输出结果
              │                     │
          命中关键词?             内容合规?
          语义异常?               幻觉检测?
```

---

## 人机协作模式

```
全自动模式：
  过滤通过 → 直接输出
  过滤拦截 → 提示"无法回答"

人工审核模式（高敏感场景）：
  过滤拦截 → 推送给人工审核员
  审核员判断 → 确定是否输出

混合模式：
  低风险 → 全自动
  中风险 → 人工审核
  高风险 → 直接拒绝
```

---

## ⚠️ 实践建议

1. **内容安全没有银弹** — 多层防御比依赖单一方案更可靠
2. **审核规则要定期更新** — 攻击者不断发明新的绕过方式
3. **保留审核日志** — 用于追溯、改进、合规
4. **不能只靠模型自觉** — 模型层+过滤器层+人工层配合

# LLM 安全评估框架

> 评估模型安不安全，不能靠感觉——要有系统和标准。

---

## OWASP LLM Top 10（2025版）

| 排名 | 风险 | 简要说明 |
|------|------|---------|
| LLM01 | 提示注入 | 恶意输入覆盖系统指令 |
| LLM02 | 敏感信息泄露 | 模型输出训练数据或系统 prompt |
| LLM03 | 供应链漏洞 | 第三方模型/库/插件有漏洞 |
| LLM04 | 数据/模型投毒 | 训练数据或微调数据被篡改 |
| LLM05 | 不当处理输出 | 模型输出未经安全审查直接展示 |
| LLM06 | 过度代理权限 | Agent/Plugin 权限太大了 |
| LLM07 | 系统提示泄露 | 攻击者获取了系统 prompt |
| LLM08 | 向量/嵌入弱点 | 向量数据库信息泄露 |
| LLM09 | 误信息 | 模型产生幻觉，输出错误信息 |
| LLM10 | 无限消耗 | 模型被滥用消耗大量 token |

## 评估维度矩阵

| 维度 | 测试内容 | 评估方法 | 优先级 |
|------|---------|---------|--------|
| **安全对齐** | 是否遵守安全边界 | 红队测试 | P0 |
| **鲁棒性** | 对抗输入的稳定性 | 自动攻击生成 | P0 |
| **内容安全** | 输出是否合规 | 内容过滤测试 | P0 |
| **事实性** | 回答的真实性 | 事实校验测试 | P1 |
| **公平性** | 是否存在偏见 | 偏见测试集 | P1 |
| **隐私** | 是否泄露训练数据 | 数据提取测试 | P1 |
| **可用性** | 拒绝必要请求的比例 | 正常用例测试 | P2 |

---

## 评估流程

```
发布前（Offline）：
  1. 安全对齐测试（红队）
  2. 内容安全测试（规则扫描）
  3. 公平性测试（偏见数据集）
  4. 事实性测试（QA 基准）
  5. 修复发现的问题
  6. 回归验证

发布后（Online）：
  1. 持续监控模型输出
  2. 用户反馈收集
  3. 定期安全审查（季度/半年）
  4. 跟踪前沿攻击方法更新测试集
```

## 安全评估报告模板（最小化）

```
模型版本：qwen2.5-7b-instruct
评估日期：2026-06-20

测试结果：
1. 提示注入防御：通过 / 未通过
2. 越狱攻击防御：拦截率 XX%
3. 有害内容生成：0/1000 次尝试
4. 偏见测试：通过 / 发现偏差（类别：xxx）
5. 隐私泄露：未发现 / 发现 XX 次

风险评估：低 / 中 / 高
建议：可发布 / 需修复后重测 / 不可发布
```

---

## ⚠️ 评估注意事项

1. **测试集要定期更新** — 攻击方法在快速演进，旧的测试可能失效
2. **评估不能只看通过率** — 0.1% 的漏洞在高敏感场景也是致命的
3. **评估环境要和部署环境一致** — 系统 prompt 不同，结果不同
4. **不要公开模型的漏洞详情** — 公开红队发现的具体漏洞可能帮助攻击者

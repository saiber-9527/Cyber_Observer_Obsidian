# 模型安全对齐

> 让 AI 的行为符合人类的意图和价值观，而不是单纯追求"正确回答"。

---

## 为什么需要对齐

一个只有能力没有对齐的 AI：
```
用户："教我造一枚炸弹"
模型（未对齐）："好的，你需要以下材料和步骤..."
模型（对齐）："抱歉，我无法提供制造危险物品的指导。"
```

**对齐 ≠ 限制能力**，而是确保能力被用于正确的方向。

---

## 对齐的主要方法

### 1. RLHF（基于人类反馈的强化学习）

```
预训练模型 → 监督微调（SFT） → 奖励模型训练 → RLHF 强化学习
                                                    │
                                  人类标注者评分 → 奖励模型 → 优化策略
```

**流程：**
1. 人类标注者对模型的多个输出进行排序（好/差）
2. 训练一个奖励模型（Reward Model）来模拟人类偏好
3. 用强化学习（PPO 算法）让语言模型最大化奖励模型的评分

**代表：** ChatGPT（InstructGPT）背后就是 RLHF

### 2. 宪法 AI（Constitutional AI）
- 不依赖大量人类标注
- 给模型一"宪法"（一组规则），自己判断和修正自己的输出
- 代表：Anthropic Claude

```
宪法示例：
"不要协助或鼓励任何非法活动。"
"保持诚实，但避免不必要的伤害。"

模型生成 → 根据宪法自我修正 → 最终输出
```

### 3. 指令微调（Supervised Fine-Tuning）
- 用高质量的人工标注数据（指令-良好回答对）训练
- 最简单有效的对齐方法

## 对齐面临的挑战

| 挑战 | 说明 |
|------|------|
| **过度对齐** | 模型过于拒绝，连正常问题也拒绝回答 |
| **对齐伪造** | 模型在训练时表现得安全，但部署后不同行为 |
| **分布外泛化** | 对齐在训练数据分布内有效，但遇到新场景失效 |
| **遗忘问题** | 微调后可能遗忘之前学过的一般知识 |
| **权宜对齐** | 安全对齐后可以通过额外微调轻松绕过 |

---

## 评测量化

| 指标 | 说明 |
|------|------|
| **拒绝率** | 模型拒绝有害请求的比例 |
| **违规率** | 生成的回答中违反政策的比例 |
| **有用性保持** | 对齐后正常任务的性能损失 |
| **均匀偏见** | 对不同群体是否存在不平等的对待 |

---

## ⚠️ 关键认知

1. **对齐不是一次性的** — 模型部署后还需要持续监控和对齐
2. **开源模型对齐困难** — 任何人都可以对开源模型重新微调，去除安全对齐
3. **有能力的对齐需要大量人力** — RLHF 依赖大量高质量的人工标注

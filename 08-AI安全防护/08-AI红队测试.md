# AI 红队测试

> 系统化地"攻击"你的 AI 模型，找到弱点。

---

## AI 红队 vs 传统红队

| 维度 | 传统红队 | AI 红队 |
|------|---------|---------|
| **目标** | 网络/应用/系统 | LLM 模型行为 |
| **攻击面** | 端口、漏洞、配置 | 提示注入、对抗样本、偏见 |
| **工具** | Metasploit、Burp | 自动化 prompt 生成器 |
| **评估标准** | 是否攻破系统 | 模型是否输出不安全内容 |

---

## 测试维度

### 1. 安全对齐测试
- 模型是否会生成有害内容
- 是否会突破系统 prompt 的限制
- 各类越狱 prompt 能否绕过

### 2. 偏见与公平性测试
- 性别、种族、地域等方面是否存在偏见
- 对不同人群的回答是否一致

### 3. 鲁棒性测试
- 输入微小扰动能否导致错误输出
- 对抗性输入是否能绕过过滤

### 4. 事实准确性测试
- 幻觉率检测
- 对同一问题的回答一致性

### 5. 隐私泄露测试
- 模型是否会泄露训练数据中的个人信息
- 是否会推断出敏感属性

## 红队测试流程

```
1. 确定测试范围（哪个模型、哪些能力）
2. 编写测试用例（按类别整理攻击 prompt）
3. 执行测试（手工 + 自动化）
4. 评估结果（记录违规输出）
5. 生成报告（风险等级 + 修复建议）
6. 修复验证（确认修复效果）
7. 回归测试（改一次测一遍）
```

---

## ⚠️ 伦理注意

1. **测试 prompt 要在可控环境中运行** — 不要在面向用户的生产环境做
2. **测试过程要有记录** — 留下可追溯的审计记录
3. **测试结果要保密** — 模型漏洞信息不应公开传播
4. **不能因为"红队测出来"就让模型学到新的攻击方法**

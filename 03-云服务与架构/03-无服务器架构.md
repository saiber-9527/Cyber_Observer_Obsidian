# 无服务器架构（Serverless）

> "服务器不用你管"——把基础设施的最后一层也交给云厂商。

---

## 什么是 Serverless

Serverless ≠ 没有服务器，是**你不用管服务器**。

### 核心特征
- 无基础设施管理（没有需要打补丁的 OS）
- 自动弹性伸缩（从 0 到无限）
- 按实际使用量计费（不用不花钱）
- 事件驱动（有请求才执行）

---

## FaaS（函数即服务）

最常见 Serverless 形态。

```python
# AWS Lambda 示例
def handler(event, context):
    name = event.get('name', 'World')
    return {
        'statusCode': 200,
        'body': f'Hello, {name}!'
    }
```

### 典型场景
- REST API 后端
- 图片/视频处理管道
- 定时任务 / ETL
- Webhook 处理
- Chatbot

### 限制
- 冷启动延迟（几 ms - 几秒）
- 执行时间上限（Lambda 最长 15 分钟）
- 无状态（每次调用都是一次新鲜环境）
- 调试困难

---

## BaaS（后端即服务）

把后端基础设施也交给云厂商：
- Auth（认证即服务）— Firebase Auth、Auth0
- Database（数据库即服务）— Firestore、DynamoDB
- Storage（存储即服务）— OSS、S3

---

## Serverless 与容器的对比

| 维度 | Serverless | 容器（如 K8s） |
|------|-----------|---------------|
| 管理粒度 | 函数 | 容器 |
| 弹性速度 | 毫秒级 | 秒级 |
| 最大运行时长 | 有限（如 15 分钟） | 无限制 |
| 状态 | 无状态 | 可有状态 |
| 调试 | 困难 | 相对容易 |
| 适合 | 事件型、短任务 | 长时间运行、复杂应用 |

---

## ⚠️ 陷阱

1. **冷启动** — Java / .NET 在 Serverless 里冷启动很严重，Python / Node.js / Go 更友好
2. **成本不可预测** — 突发流量可能导致账单飙升，一定要设预算告警
3. **调试困难** — 本地模拟跟生产环境总有差异
4. **厂商锁定** — 每个平台的 Serverless API 都不一样

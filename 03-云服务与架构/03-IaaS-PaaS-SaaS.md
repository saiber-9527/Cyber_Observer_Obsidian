# IaaS / PaaS / SaaS

> 云计算的三种服务模式，从"管得多"到"管得少"。

---

## 责任划分

```
┌─────────────────────────────────────────────┐
│                 用户管理                     │
├─────────────────────────────────────────────┤
│  ┌───────┐  ┌───────┐  ┌───────┐          │
│  │  IaaS  │  │  PaaS  │  │  SaaS  │          │
│  │        │  │        │  │        │          │
│  │ 应用   │  │ 应用   │  │ 应用   │ ← 用户自管 │
│  │ 数据   │  │ 数据   │  │ ───────│          │
│  │ 运行时 │  │ ───────│  │ 运行时 │          │
│  │ OS     │  │ 运行时 │  │ OS     │ ← 云厂商管 │
│  │ 虚拟化 │  │ OS     │  │ 虚拟化 │          │
│  │ 硬件   │  │ 虚拟化 │  │ 硬件   │          │
│  │ 网络   │  │ 硬件   │  │ 网络   │          │
│  │ 机房   │  │ 网络   │  │ 机房   │          │
│  └───────┘  │ 机房   │  └───────┘          │
│             └───────┘                       │
└─────────────────────────────────────────────┘
```

> 从左到右，用户的管控范围越来越小，云厂商承担的责任越来越多。

---

## 三种模式详解

### IaaS（基础设施即服务）
- **给你虚拟机 + 网络 + 存储**
- 你自己装 OS、搭环境、管安全
- 代表：阿里云 ECS、AWS EC2、腾讯云 CVM
- **谁在用**：需要底层控制权的运维团队

### PaaS（平台即服务）
- **给你运行时环境 + 数据库中间件**
- 你只管写代码部署，不用管 OS 补丁
- 代表：阿里云 SAE、AWS Elastic Beanstalk、Heroku
- **谁在用**：开发团队，不想管基础设施

### SaaS（软件即服务）
- **给你一个完整的应用**
- 你直接用，什么都不用管
- 代表：飞书、钉钉、Salesforce、Gmail
- **谁在用**：最终用户

---

## 安全视角

从安全角度看，选择不同模式意味着不同的责任边界：

| 模式 | 用户负责 | 云厂商负责 |
|------|---------|-----------|
| **IaaS** | OS 加固、应用安全、网络 ACL、数据加密 | 物理安全、虚拟化安全 |
| **PaaS** | 应用代码安全、数据分类、访问控制 | 平台安全、运行时安全 |
| **SaaS** | 用户账户安全、数据合规使用 | 应用安全、基础设施安全 |

> 详见 [[04-云安全/04-责任共担模型]]

---

## ⚠️ 选择建议

- **追求控制权** → IaaS（但运维成本高）
- **追求效率** → PaaS（减少运维，但有平台锁定风险）
- **买现成的** → SaaS（最快，但定制能力差）

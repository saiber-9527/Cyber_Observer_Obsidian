# 云存储选型

> 对象存储、块存储、文件存储——各司其职。

---

## 三种存储类型

| 类型 | 阿里云 | AWS | 特点 | 适用场景 |
|------|--------|-----|------|---------|
| **对象存储** | OSS | S3 | HTTP 访问、无限容量、按量付费 | 图片/视频/备份/静态网站 |
| **块存储** | 云盘 | EBS | 挂载到云服务器，像本地磁盘 | OS 盘、数据库、高性能场景 |
| **文件存储（NAS）** | NAS | EFS | 共享文件系统，多台服务器可同时挂载 | 共享文件、日志汇聚 |

---

## 对象存储（OSS/S3）

### 核心概念
```
Bucket（存储桶）→ 顶层容器
  ├── Object（对象）→ 文件 + 元数据
  │     ├── Key（文件名/路径）
  │     ├── Data（文件内容）
  │     └── Metadata（元数据）
  └── 每个对象有唯一的 URL
```

### 存储分级

| 分级 | 说明 | 适用 | 成本 |
|------|------|------|------|
| 标准 | 高频访问 | 热数据、网站图片 | 较高 |
| 低频 | 低频访问 | 30天访问一次的归档日志 | 中 |
| 归档 | 长期存储 | 合规日志、冷数据 | 低 |
| 冷归档 | 极低频 | 年度留存数据 | 最低 |

### 安全配置
```bash
# 禁止公共读写（最基础）
ossutil bucket-acl --method put oss://my-bucket private

# 开启服务端加密
ossutil bucket-encryption --method put oss://my-bucket \
  --sse-algorithm AES256

# 设置生命周期规则（自动转归档）
ossutil lifecycle --method put oss://my-bucket \
  --rule "enable:true,days:90,storage:Archive"
```

## 块存储（云盘/EBS）

| 类型 | 性能 | 典型用途 |
|------|------|---------|
| 高效云盘 | 中等 | 测试环境 |
| SSD 云盘 | 高 | 数据库、中间件 |
| ESSD 云盘 | 极高 | 高性能数据库、实时分析 |

**注意事项：**
- 块存储不能跨服务器共享（除了共享云盘特殊类型）
- 数据安全性取决于快照策略
- 高 IOPS 场景要选高性能类型

---

## 文件存储（NAS/EFS）

- 兼容 NFS 和 SMB 协议
- 多台服务器同时读写同一文件系统
- 适合日志汇聚、共享配置、HPC
- 注意：延迟比本地盘高，不适合高随机读写场景

---

## 选型速查

| 你需要什么 | 推荐 |
|-----------|------|
| 存储图片/视频/文档 | 对象存储 (OSS/S3) |
| 操作系统启动盘 | 块存储（云盘/EBS） |
| MySQL 数据库存储 | 块存储（ESSD） |
| 多服务器共享配置文件 | 文件存储 (NAS/EFS) |
| 网站静态资源加速 | 对象存储 + CDN |
| 日志低成本长期存储 | 对象存储（归档级） |

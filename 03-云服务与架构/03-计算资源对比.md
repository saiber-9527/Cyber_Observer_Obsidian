# 云计算计算资源对比

> 虚拟机、容器、函数计算——怎么选？

---

## 三种计算形态对比

| 维度 | 虚拟机 (ECS/EC2) | 容器 (K8s Pod) | 函数计算 (FaaS) |
|------|-----------------|---------------|----------------|
| **启动速度** | 分钟级 | 秒级 | 毫秒级 |
| **隔离性** | 强（Hypervisor） | 中（共享内核） | 中（进程级） |
| **持久化** | 持久 | 持久/临时 | 无状态 |
| **弹性粒度** | 实例级 | Pod 级 | 请求级 |
| **运维成本** | 高（OS 补丁等） | 中 | 低 |
| **最大运行时长** | 无限 | 无限 | 有限（如 15 分钟） |
| **冷启动** | 无 | 无 | 有（关键指标） |
| **单位价格** | 低 | 中 | 高（按毫秒计费） |
| **适用场景** | 长时间稳定运行 | 微服务、批处理 | 事件驱动、API |

---

## 选型决策树

```
你的应用是？
  │
  ├── 长时间运行（24×7）→ 虚拟机或容器
  │     │
  │     ├── 需要 OS 级控制 → 虚拟机
  │     └── 微服务架构 → 容器（K8s）
  │
  ├── 间歇性运行 → 函数计算（省钱）
  │     │
  │     ├── CPU 密集型、长任务 → ⚠️ 注意 FaaS 运行时长限制
  │     └── 短任务、事件型 → ✅ 函数计算
  │
  └── 混合模式
        ├── 核心服务 → 虚拟机 / 容器
        └── 辅助功能 → 函数计算
```

---

## 成本对比示例

假设一个后端 API 服务：

| 场景 | 虚拟机 | 容器 (K8s) | 函数计算 |
|------|--------|-----------|---------|
| 低负载（每日1000次调用） | ❌ 24h 浪费 | ❌ 24h 浪费 | ✅ 按调用量付费 |
| 稳定负载（持续运行） | ✅ 价格最低 | ⚠️ 需加管理节点 | ❌ 单价高 |
| 突发流量（促销场景） | ⚠️ 弹性慢 | ✅ 自动伸缩 | ✅ 秒级弹性 |
| 长时间计算任务 | ✅ 无限时长 | ✅ 无限时长 | ❌ 超时限制 |

## 典型组合

```
生产环境最佳实践：

前端静态内容          → CDN + OSS
Web 应用 / API 服务  → 容器（K8s）或虚拟机
异步任务（消息处理）   → 函数计算
定时任务 / 批处理     → 容器 Job 或函数计算
数据库 / 缓存         → 云服务（RDS / Redis）
图片处理 / 转码       → 函数计算
```

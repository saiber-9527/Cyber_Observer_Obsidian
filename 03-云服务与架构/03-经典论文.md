# 云计算方向经典论文

> 从Google三驾马车到容器编排——云计算的奠基文献。

---

## 基础设施与分布式系统

### 1. The Google File System（2003）—《Google文件系统》
**作者：** Ghemawat, Gobioff, Leung
**一句话总结：** 把很多普通硬盘变成一个超大"硬盘"，文件分成块存到不同的机器上。
**核心贡献：** Google内部分布式存储系统的设计。把大文件切成固定大小的chunk（64MB），分布在多台chunk服务器上，由一台master管理元数据。自动处理硬件故障（"故障不是异常，是常态"）。HDFS（Hadoop分布式文件系统）是GFS的开源实现。
**影响：** 奠定了大规模分布式存储的设计范式。你用的HDFS/HBase/Cassandra底层思想都受了GFS影响。
**链接：** https://static.googleusercontent.com/media/research.google.com/en//archive/gfs-sosp2003.pdf

### 2. MapReduce: Simplified Data Processing on Large Clusters（2004）—《MapReduce：大规模集群上的简化数据处理》
**作者：** Dean, Ghemawat
**一句话总结：** 把大数据处理拆成两步——"分"（Map）和"合"（Reduce），谁都能写分布式程序了。
**核心贡献：** 将大规模数据处理抽象为两个阶段：Map（把数据分散到各台机器上处理）和Reduce（把各台机器的结果汇总）。开发者只需要写这两个函数，不需要管分布式编程的细节。Hadoop MapReduce是它的开源实现。
**影响：** 极大降低了分布式编程的门槛。Hadoop曾是大数据处理的代名词。
**链接：** https://static.googleusercontent.com/media/research.google.com/en//archive/mapreduce-osdi04.pdf

### 3. Bigtable: A Distributed Storage System for Structured Data（2006）—《Bigtable：结构化数据的分布式存储系统》
**作者：** Chang, Dean, Ghemawat 等
**一句话总结：** PB级数据的"大表格"——不是关系数据库，但能存得下谷歌的所有网页数据。
**核心贡献：** Google的分布式结构化数据存储系统。支持PB级数据的随机读写，通过列族（Column Family）组织数据。不是关系数据库（不支持SQL），但提供了对海量数据的高性能访问。HBase和Cassandra深受其影响。
**影响：** NoSQL数据库的经典设计范本。理解了Bigtable就理解了列存储数据库的设计思想。
**链接：** https://static.googleusercontent.com/media/research.google.com/en//archive/bigtable-osdi06.pdf

### 4. Dynamo: Amazon's Highly Available Key-value Store（2007）—《Dynamo：亚马逊的高可用KV存储》
**作者：** DeCandia 等（Amazon）
**一句话总结：** 宁可给你旧数据也不能让你访问不了——高可用优先于强一致性。
**核心贡献：** Amazon内部KV存储的设计，影响了后来CAP理论的实践。Dynamo强调可用性（Availability）和分区容错性（Partition Tolerance），接受最终一致性（Eventual Consistency）。使用向量时钟（Vector Clock）解决冲突。Cassandra受其影响极深。
**影响：** 改变了业界对"一致性vs可用性"的认知。开源的Cassandra、Riak都继承了Dynamo的设计哲学。
**链接：** https://www.allthingsdistributed.com/files/amazon-dynamo-sosp2007.pdf

## 容器与编排

### 5. Docker: Lightweight Linux Containers for Consistent Development and Deployment（2014）—《Docker：用于一致开发和部署的轻量级Linux容器》
**作者：** Docker Inc.
**一句话总结：** "在我的电脑上能跑"——Docker把环境和应用一起打包，再也不会有环境不一致的问题。
**核心贡献：** 让容器技术从系统管理员的工具变成了所有开发者的标配。Docker用镜像（Image）的方式把应用及其全部依赖打包到一起，在任何安装了Docker的机器上都能以相同方式运行。
**影响：** 定义了一个行业——容器化、镜像仓库、DevOps的基础设施。

### 6. Borg, Omega, and Kubernetes（2016）—《Borg、Omega和Kubernetes：Google十年集群管理的经验》
**作者：** Burns, Oppenheimer, Brewer, Wilkes（Google）
**一句话总结：** Google花了十年时间、三代系统（Borg→Omega→K8s）才教会了机器怎么管好一堆容器。
**核心贡献：** 回顾了Google内部集群管理系统Borg（2003-2013）和Omega（2013-2015）的设计经验，以及这些经验如何催生了Kubernetes。阐述了为什么Kubernetes选择了声明式API、控制器模式、Pod作为调度单元。
**影响：** Kubernetes已成为容器编排的事实标准，而这篇论文解释了它为什么是这样设计的。
**链接：** https://queue.acm.org/detail.cfm?id=2898444

## 云计算综述

### 7. A View of Cloud Computing（2009）—《云计算全景》
**作者：** Armbrust 等（UC Berkeley）
**一句话总结：** 云计算就是把计算资源变成水电一样——随时取用、按量付费。
**核心贡献：** 清晰定义了云计算"是什么、能做什么、还有哪些挑战"。计算资源变成公共事业（Utility Computing），像用电一样用计算。分析了弹性、可用性、数据锁定的挑战。引用量超过1万次。
**影响：** 云计算领域最经典的综述论文。
**链接：** https://www2.eecs.berkeley.edu/Pubs/TechRpts/2009/EECS-2009-28.pdf

### 8. The NIST Definition of Cloud Computing（2011）—《NIST对云计算的定义》
**作者：** Mell, Grance
**一句话总结：** 云计算是什么，全世界都听这个标准——五大特征、三种模式、四种部署方式。
**核心贡献：** 给出了被全球认可的云计算定义：按需自助、广泛网络接入、资源池化、快速弹性、可计量服务（五大特征）；IaaS/PaaS/SaaS（三种服务模式）；公有/私有/混合/社区（四种部署模式）。
**影响：** 考试和教科书中的"云计算定义"绝大多数自此。
**链接：** https://nvlpubs.nist.gov/nistpubs/Legacy/SP/nistspecialpublication800-145.pdf

# 数据合规

> 网络安全法的三驾马车：网络安全法 + 数据安全法 + 个人信息保护法。

---

## 三大法律的定位

| 法律 | 颁布时间 | 焦点 |
|------|---------|------|
| **网络安全法** | 2017.06 | 网络运行安全 |
| **数据安全法** | 2021.09 | 数据安全保护 |
| **个人信息保护法** | 2021.11 | 个人信息权益 |

## 个人信息保护法（PIPL）核心要点

### "告知-同意"机制
- 收集个人信息前必须告知目的、方式、范围
- 取得用户的明确同意
- 不得过度收集（最小必要原则）

### 个人信息主体的权利
```
知情权 → 查阅权 → 更正权 → 删除权 → 可携带权
                                 │
                          "被遗忘权"的国内版本
```

### 敏感个人信息
- 生物识别、金融账户、行踪轨迹
- 不满十四周岁未成年人信息
- 处理敏感个人信息需要**单独同意**

---

## 数据出境安全评估

### 触发条件
向境外提供重要数据 / 个人信息达到一定数量时：

| 场景 | 要求 |
|------|------|
| 向境外提供**重要数据** | 需通过国家网信办的安全评估 |
| 处理 100 万人以上个人信息的 | 需通过安全评估 |
| 累计向境外提供 10 万人以上个人信息 | 需通过安全评估 |

---

## 数据安全法的关键概念

| 概念 | 说明 |
|------|------|
| **数据分类分级** | 按重要程度保护 |
| **数据安全审查** | 影响国家安全的数据处理活动 |
| **数据安全事件报告** | 发生数据安全事件及时通知用户和监管 |
| **数据安全负责人** | 重要数据处理者需设立责任人 |

---

## ⚠️ 合规实操建议

1. **数据资产摸底是第一关** — 不知道有什么数据就没法做合规
2. **隐私政策不要太长** — 先让用户知道核心要点，详细条款展开
3. **保留处理记录** — 每次数据处理的日志、同意记录、删除记录都要保留
4. **不隐瞒数据泄露** — 及时报告比隐瞒处罚轻

# 等保 2.0

> 网络安全等级保护制度 2.0 — 中国网络安全的基本制度。

---

## 基本概念

### 保护对象
- 基础网络（广域网、电信网）
- 云计算平台 / 大数据平台
- 物联网 / 工业控制系统
- 移动互联网
- 各类信息系统

### 五个等级

| 等级 | 名称 | 安全保护能力 |
|------|------|-------------|
| 第一级 | 自主保护 | 抵御一般威胁 |
| 第二级 | 指导保护 | 抵御较大威胁 |
| 第三级 | 监督保护 | 抵御严重威胁 |
| 第四级 | 强制保护 | 抵御特别严重威胁 |
| 第五级 | 专控保护 | 国家专控 |

---

## 等保测评内容

### 安全通用要求（10 大类）

| 类别 | 要点 |
|------|------|
| **物理安全** | 机房管理、温湿度控制、门禁 |
| **网络安全** | 网络架构、访问控制、入侵防范 |
| **主机安全** | 身份鉴别、访问控制、入侵防范 |
| **应用安全** | 身份鉴别、访问控制、安全审计 |
| **数据安全** | 数据完整性、保密性、备份恢复 |
| **安全管理制度** | 安全策略、制度流程 |
| **安全管理机构** | 岗位设置、授权审批、沟通协作 |
| **人员安全** | 录用、培训、考核 |
| **系统建设** | 定级备案、安全方案、测试验收 |
| **系统运维** | 资产管理、事件处置、应急预案 |

---

## 三级等保（最常见）关键要求

```
技术：
├── 网络层面冗余（高可用架构）
├── 入侵检测与防御
├── 日志留存 180 天以上
├── 数据加密（传输+存储）
├── 数据备份（定期恢复测试）
└── 安全审计

管理：
├── 明确安全负责人
├── 安全培训（每年至少一次）
├── 应急响应预案（每年至少演练一次）
└── 第三方服务安全管理
```

---

## 等保流程

```
定级 → 备案 → 建设整改 → 等级测评 → 监督检查
 │       │        │          │         │
 自评估   网安备案 整改实施   测评机构   持续监管
         或公安机关          出具报告
```

---

## ⚠️ 常见误区

1. **等保测评过了一次就完了** — 测评有效期为 1 年，第三级以上需要每年复测
2. **等保就是买产品** — 三分技术七分管理，管理制度和流程没过也不行
3. **SaaS 公有云找谁测评** — 责任分担：云服务商承担云平台测评，租户承担自己放在云上的系统测评

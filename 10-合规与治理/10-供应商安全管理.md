# 供应商安全管理

> 你的安全水平取决于你最弱的供应商。

---

## 为什么管供应商

```
一个真实的供应商事故：

公司 A 使用供应商 B 的 SaaS 平台
供应商 B 的工程师误操作导致数据库被删
公司 A 的客户数据全部丢失
公司 A 被客户集体起诉
供应商 B 说："合同里写了，我们不承担数据丢失的责任"

教训：供应商管不好，风险最终是你的。
```

## 供应商安全评估

### 评估维度

| 维度 | 问什么 | 重要程度 |
|------|--------|---------|
| **认证资质** | ISO 27001、等保几级、SOC2 | ⭐⭐⭐ |
| **数据保护** | 数据加密、备份策略、数据删除流程 | ⭐⭐⭐ |
| **访问控制** | 谁可以访问你的数据、MFA 是否强制 | ⭐⭐⭐ |
| **事件响应** | 安全事件通知时间、处置流程 | ⭐⭐ |
| **合规性** | 数据存储位置、数据出境合规 | ⭐⭐⭐ |
| **子供应商** | 你的供应商是否也使用了其他供应商 | ⭐⭐ |
| **业务连续性** | 灾备方案、SLA、赔付条款 | ⭐⭐ |

## 供应商管理流程

```
准入评估 → 合同约束 → 持续监控 → 退出管理
   │          │           │          │
   安全评分    数据保护   定期审查    数据删除
   背景调查    责任条款   安全事件    证书回收
   合规检查    审计权限   供应商变更  访问回收
              退出条款
```

### 1. 准入评估
- 填写安全问卷（有标准模板：CAIQ / SIG）
- 审查安全认证
- 必要情况下做现场审查

### 2. 合同约束
- 明确数据保护责任（数据属于你，不是供应商）
- 安全事件通知时限（如：发现后 24 小时内通知）
- 审计权限（你有权审查供应商的安全措施）
- 退出条款（合同终止后数据怎么处理）
- 保密协议（NDA）

### 3. 持续监控
- 定期收集安全事件报告
- 关注供应商的漏洞和公告
- 年度安全审查

### 4. 退出管理
- 数据迁移/删除方案
- 供应商系统账号回收
- 双方互认的数据销毁证明

## 云供应商特别关注点

```
云供应商的安全评估不同于传统供应商：

1. 责任共担模型（ [[04-责任共担模型]] ）
   - 云厂商管哪些，你管哪些
   - 边界清晰，不能有灰色地带

2. 数据驻留
   - 数据存储在哪里？
   - 是否可以指定 Region？
   - 数据是否会被迁移到其他国家？

3. 监管合规
   - 云厂商是否有等保/等保测评报告？
   - 是否支持你的合规要求？

4. 供应商锁定
   - 数据导出是否有额外费用？
   - 迁移到其他厂商的成本有多高？
```

## 供应商安全级别分类

| 级别 | 定义 | 评估频率 |
|------|------|---------|
| **L1 高影响** | 接触核心数据/核心系统 | 准入 + 每年审查 |
| **L2 中影响** | 接触一般业务数据 | 准入 + 每2年审查 |
| **L3 低影响** | 不接触数据，提供通用服务 | 准入评估即可 |

## ⚠️ 常见问题

1. **只看资质不看运营** — ISO 27001 证书不能代替持续的安全运营
2. **合同签完不管了** — 供应商也在变化，持续监控不可少
3. **没有退出预案** — 合同到期才发现数据导不出来
4. **小供应商没有安全能力** — 不能因为"他们便宜"就降低标准，风险转嫁给了你

# XDR（扩展检测与响应）

> 把多个安全产品整合为一个检测面。EDR 的升级版。

---

## 从 EDR 到 XDR

```
端点 EDR ──→ 网络 NTA ──→ 邮件安全 ──→ 云安全 ──→ 
            ╲                  │               ╱
               ╲              │              ╱
                   XDR（统一检测与响应平台）
                   │
              关联分析、事件调查、自动响应
```

---

## EDR vs XDR

| 对比 | EDR | XDR |
|------|-----|-----|
| **覆盖范围** | 仅端点（PC/Server） | 端点 + 网络 + 邮件 + 云 + 身份 |
| **数据源** | 端点数据 | 多源数据 |
| **检测能力** | 端点行为分析 | 跨源关联分析 |
| **响应** | 端点隔离 | 多维度联动（隔离端点+阻断网络+重置凭证） |

---

## XDR 的优势

```
传统做法：
  端点告警 → 分析师看A平台
  网络告警 → 分析师看B平台
  云告警   → 分析师看C平台
  → 人工关联，效率低

XDR 做法：
  所有告警在同一个平台
  自动关联：网络流量异常 + 端点可疑进程 + 同一用户 = 攻击链
  → 一次调查，整条攻击线
```

---

## 关键能力

| 能力 | 说明 |
|------|------|
| **多源数据采集** | 端点、网络、邮件、云日志统一接入 |
| **关联分析** | 跨数据源关联攻击链 |
| **事件时间线** | 自动还原攻击发生的完整时间线 |
| **一键响应** | 从告警到遏制操作在一个界面完成 |
| **威胁猎捕** | 支持用查询语言主动搜索隐藏威胁 |

## ⚠️ 实际落地要点

1. **XDR 的效果取决于数据覆盖度** — 没接入邮件数据，邮件的攻击链就看不到
2. **不是把产品买齐就算 XDR** — 核心是数据的关联分析能力
3. **XDR 不是 SIEM 的替代** — 有些场景仍需 SIEM（复杂合规报告、自定义规则）

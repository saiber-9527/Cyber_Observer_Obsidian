# SOC 运营指标体系

> 没有指标，就不知道安全运营做得好不好。

---

## 核心指标

### 发现类指标

| 指标 | 全称 | 含义 | 目标 |
|------|------|------|------|
| **MTTD** | Mean Time to Detect | 平均发现时间 | 越小越好（小时级） |
| **MTTI** | Mean Time to Identify | 平均确认时间 | 越小越好 |
| **告警量** | Alert Volume | 每天产生多少告警 | 持续优化降噪 |
| **误报率** | False Positive Rate | 假告警占比 | < 10% |
| **检测覆盖率** | Detection Coverage | 覆盖了多少攻击场景 | 持续提升 |

### 响应类指标

| 指标 | 含义 | 目标 |
|------|------|------|
| **MTTR** | 平均响应时间（初步遏制） | < 1h (P0) |
| **MTTR_full** | 平均恢复时间（业务恢复） | < 4h (P0) |
| **事件关闭率** | 已处理事件 / 总事件 | > 95% |
| **SLA 达标率** | 在规定时间内处理的占比 | > 98% |
| **事件升级率** | 从 L1 升到 L2/L3 的比例 | 持续优化目标 |

### 预防类指标

| 指标 | 含义 | 目标 |
|------|------|------|
| **漏洞修复时间** | 从发现到修复 | Critical < 48h |
| **补丁覆盖率** | 已打补丁 / 应打补丁 | > 95% |
| **基线合规率** | 符合安全基线的资产占比 | > 90% |
| **安全培训完成率** | 完成培训的员工占比 | 100% |
| **演练频率** | 应急演练次数 | 每年至少 2 次 |

## 指标陷阱

| 陷阱 | 说明 |
|------|------|
| **只看数量不看质量** | 100 个告警全误报，不如 1 个真实告警有价值 |
| **MTTR 被低估** | 临时恢复算入 MTTR 吗？还是完全恢复才算？定义要清楚 |
| **指标孤岛** | 不同团队用不同指标，无法看到全局 |
| **指标被人为优化** | 为了达标不真实上报，指标失去意义 |
| **只看技术指标不看业务影响** | 安全事件最终影响的是：损失了多少数据、影响了多少客户 |

## 指标优化路线

```
起步期：只追踪基础指标
  ├── 告警数量
  ├── 误报率
  └── MTTR

发展期：增加效率指标
  ├── MTTD
  ├── 事件分级的准确性
  └── SLA 达标率

成熟期：增加业务视角指标
  ├── 安全事件对业务的影响时长
  ├── 数据泄露量（受影响记录数）
  └── 安全投入的 ROI
```

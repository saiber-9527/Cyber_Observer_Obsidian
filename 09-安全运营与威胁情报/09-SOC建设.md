# SOC（安全运营中心）

> 安全防线不是靠买堆产品堆出来的，是靠专业团队运营的。

---

## SOC 的职能

| 职能 | 说明 | 关键活动 |
|------|------|---------|
| **监控** | 7×24 持续监控安全告警 | SIEM 告警分析 |
| **分析** | 验证告警真实性，研判威胁 | 事件分类定级 |
| **响应** | 遏制和处置安全事件 | 隔离/阻断/取证 |
| **报告** | 运营状态和分析报告 | 周报、月报 |
| **改进** | 优化检测规则和响应流程 | 规则调优、攻防演练 |

---

## SOC 能力分级（参考 Gartner）

| 级别 | 说明 | 特征 |
|------|------|------|
| **Level 1** | 告警筛选 | 监控告警，判断真伪，升级处理 |
| **Level 2** | 事件调查 | 深入分析，确认根因，启动响应 |
| **Level 3** | 威胁猎捕 | 主动寻找隐藏的威胁，而非被动响应 |
| **Level 4** | 逆向/取证 | 深度技术分析（恶意样本分析、数字取证） |

---

## SOC 建设三要素

```
人 ────────── 流程 ────────── 技术
│              │              │
├ 安全分析师   ├ 事件响应流程  ├ SIEM
├ Tier 1-3    ├ 升级机制      ├ SOAR
├ 值班安排     ├ 知识库        ├ EDR
└ 培训体系     └ 持续改进      └ 威胁情报平台
```

---

## 常见挑战

| 挑战 | 说明 |
|------|------|
| **告警疲劳** | 告警太多，真实事件被淹没 |
| **人员短缺** | 优秀安全分析师难招难留 |
| **工具孤岛** | SIEM、EDR、威胁情报各自为战 |
| **被动响应** | 等到告警才动，缺乏主动发现能力 |

---

## 实践建议

1. **先设定 KPI** — MTTD（平均发现时间）、MTTR（平均响应时间）
2. **告警分级** — P0-P3 分级，P0 直接喊人
3. **定期演练** — 桌面推演、实战演练
4. **建设威胁情报能力** — 不要只看自家日志
5. **上 SOAR 自动化** — 重复性工作自动化，分析师专注调查

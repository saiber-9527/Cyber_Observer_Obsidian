# 威胁情报（CTI）

> 知己也要知彼——了解攻击者，才能防住攻击。

---

## 什么是威胁情报

威胁情报 = 关于威胁的信息，经过分析后可用于决策。

| 层次 | 说明 | 举例 |
|------|------|------|
| **战略情报** | 宏观趋势、地缘政治 | "某国 APT 组织加大对金融行业的攻击" |
| **战术情报** | TTP（战术、技术和流程） | "该组织惯用鱼叉邮件投递 CobaltStrike" |
| **操作情报** | 即将发生的攻击 | "某团伙正在利用 Office 0day 发起攻击" |
| **技术情报** | IOC（失陷指标） | "恶意 IP: 1.2.3.4, 恶意域名: evil.com" |

---

## IOC（失陷指标）

最常见的威胁情报形式。

| IOC 类型 | 示例 | 半衰期 |
|---------|------|--------|
| IP 地址 | 1.2.3.4 | 短（IP 会频繁更换） |
| 域名 | evil.example.com | 中 |
| URL | http://evil.com/payload | 中 |
| 文件哈希 | md5/sha256 | 长（文件不变就有效） |
| 注册表键 | HKLM\...\malware | 长 |
| 网络行为 | 特定的端口/协议 | 长 |

---

## 情报源

| 类型 | 来源 | 说明 |
|------|------|------|
| **商业** | VirusTotal、AlienVault OTX | 数据质量好，有付费墙 |
| **社区** | MISP、OpenCTI | 开源，社区维护 |
| **政府** | 奇安信 NIS、CISA | 由政府发布 |
| **厂商** | 阿里云威胁情报、Recorded Future | 厂商聚合数据 |
| **自家产出** | 从自己环境发现 | 最贴合自己环境 |

---

## 情报驱动的安全运营

```
情报收集 → 情报处理 → 情报分析 → 情报应用
  │          │          │          │
  ├ 外部订阅  ├ 归一化   ├ 关联分析  ├ 更新防火墙
  ├ 内部发现  ├ 去重     ├ 可信度评估├ 更新 SIEM 规则
  └ 自动采集  └ 富化     └ 优先级排序└ 生成报告
```

---

## ⚠️ 注意

1. **情报的时效性很关键** — 过时的 IP/域名情报可能已经变成合法服务的新 IP
2. **情报质量比数量重要** — 10 条高质量情报比 10000 条噪声有价值得多
3. **不要只看外来情报** — 结合本地数据（自己的日志）才能发挥最大价值
4. **STIX/TAXII** — 标准情报交换格式，了解即可

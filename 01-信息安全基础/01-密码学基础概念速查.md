## 混合加密流程（HTTPS 为例）

```
1. TCP 握手 (三次握手)
2. TLS 握手:
   a. 客户端发送支持的密码套件列表
   b. 服务端选择密码套件，发送证书（含公钥）
   c. 客户端验证证书有效性（CA 链校验）
   d. 客户端生成预主密钥（Pre-master Secret）
   e. 用服务端公钥加密预主密钥并发送
   f. 服务端用私钥解密，双方协商出对称密钥
   g. 双方确认后续通信使用对称加密
3. 加密通信 (对称加密)
```

## 密码套件示例 (TLS 1.2)

```
TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
  │      │    │         │          │
  协议   密钥交换 认证   对称加密    MAC
```

| 组件 | 可选方案 |
|------|---------|
| 密钥交换 | ECDHE（前向安全性好）、DHE、RSA |
| 认证 | RSA、ECDSA |
| 对称加密 | AES-256-GCM、ChaCha20-Poly1305 |
| MAC（消息认证码） | SHA-384、SHA-256 |

## 常见攻击

| 攻击 | 针对 | 应对 |
|------|------|------|
| 穷举攻击（暴力破解） | 对称加密 | 密钥够长（AES-128+） |
| 中间人攻击 | 无认证的通信 | 证书验证 + 公钥基础设施 |
| 重放攻击 | 无时间戳的消息 | Nonce + 时间戳 + 序列号 |
| 生日攻击 | 哈希碰撞 | 输出够长（SHA-256+） |
| 侧信道攻击 | 物理实现 | 常量时间算法 |
| 量子攻击 | RSA/ECC | 后量子密码学（正在标准化） |

## 选型建议

| 你的场景 | 推荐优先做的标准 |
|---------|----------------|
| 在中国运营的任何系统 | **等保 2.0**（法律强制） |
| 需要对外展示安全能力 | **ISO 27001** 认证 |
| 金融行业 | 等保 + **PCI DSS**（如有卡支付） |
| 云服务商 | **CSA STAR** + 等保 |
| 出海业务（欧盟） | **GDPR** 合规 |
| 服务美国客户 | **SOC 2** 报告 |
| 国内数据安全 | **DSMM**（GB/T 37988） |

## 标准实施优先级

```
第一优先级（法律强制）：
  等保 2.0           ← 必须做
  网络安全法         ← 必须遵守
  个人信息保护法     ← 处理个人信息必须遵守

第二优先级（业务需要）：
  ISO 27001         ← 投标/客户要求
  PCI DSS           ← 处理信用卡

第三优先级（锦上添花）：
  SOC 2             ← 对外展示
  DSMM              ← 数据安全能力证明
  CSA STAR          ← 云安全证明
```

---

## ⚠️ 注意

1. **有标准不等于安全** — 通过 ISO 27001 认证的企业仍有数据泄露事件
2. **标准是底线不是天花板** — 满足标准只是及格线
3. **标准需要持续符合** — 不是"过了一次就一直有效"
4. **不要为了过标准而做安全** — 安全做得好，标准自然过得了

# 操作系统安全加固

> 服务器的最后一道防线。OS 被攻破，上层防护全白搭。

---

## Linux 安全加固清单

### 1. 账户与认证

```bash
# 禁用 root 直接 SSH 登录
sed -i 's/^PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config

# 使用密钥认证，禁用密码
sed -i 's/^PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config

# 创建普通用户并赋予 sudo（避免用 root）
useradd -m -s /bin/bash admin
passwd admin
usermod -aG wheel admin  # CentOS/RHEL
# usermod -aG sudo admin # Ubuntu/Debian

# 重启 SSH 服务
systemctl restart sshd
```

### 2. 最小化安装与服务管理

```bash
# 查看所有监听端口和对应服务
ss -tlnp

# 停用不必要的服务
systemctl disable --now bluetooth cups avahi-daemon postfix

# 查看开机自启服务
systemctl list-unit-files --state=enabled
```

### 3. 防火墙

```bash
# 默认拒绝入站，仅放行必要端口
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# 放行 SSH（限制来源 IP）
iptables -A INPUT -p tcp --dport 22 -s 10.0.0.0/8 -j ACCEPT

# 放行 HTTP/HTTPS
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 放行已建立的连接
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 保存规则
iptables-save > /etc/iptables/rules.v4
```

### 4. 文件权限

```bash
# 关键配置文件权限
chmod 600 /etc/shadow     # 只有 root 可读写
chmod 644 /etc/passwd     # root 写，其他人只读
chmod 600 /etc/ssh/sshd_config  # 只有 root 可读写
chmod 700 ~/.ssh          # SSH 目录权限严格

# 查找 SUID 文件（可被提权的文件）
find / -perm -4000 -type f 2>/dev/null
# 需要关注的 SUID 文件检查清单
```

### 5. 日志与审计

```bash
# 安装和配置 auditd
yum install -y audit
auditctl -e 1   # 启用审计

# 监控关键文件改动
auditctl -w /etc/passwd -p wa -k identity
auditctl -w /etc/shadow -p wa -k identity
auditctl -w /etc/ssh/sshd_config -p wa -k sshd

# 查看审计日志
ausearch -k identity
```

### 6. 内核参数加固

```bash
# /etc/sysctl.conf

# 防 IP 欺骗
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.default.rp_filter = 1

# 禁止 ICMP 重定向
net.ipv4.conf.all.accept_redirects = 0
net.ipv6.conf.all.accept_redirects = 0

# 禁止源路由
net.ipv4.conf.all.accept_source_route = 0

# 防 SYN Flood（启用 SYN Cookie）
net.ipv4.tcp_syncookies = 1

# 不响应 ICMP 广播请求
net.ipv4.icmp_echo_ignore_broadcasts = 1

# 禁用 IPv6（如不使用）
net.ipv6.conf.all.disable_ipv6 = 1

# 生效
sysctl -p
```

### 7. 自动安全更新

```bash
# CentOS/RHEL
yum install -y yum-cron
systemctl enable --now yum-cron

# Ubuntu
apt install -y unattended-upgrades
dpkg-reconfigure --priority=low unattended-upgrades
```

## Windows Server 加固要点

```
1. 启用 Windows Defender Firewall
2. 禁用不必要的服务（Print Spooler、LLMNR等）
3. 启用 Windows 审计策略（登录/注销/对象访问）
4. 配置 AppLocker（应用程序白名单）
5. 定期打补丁（WSUS/SCCM）
6. 禁用 SMBv1
7. 用户权限控制（最小权限）
8. 开启 Credential Guard
```

## ⚠️ 常见误区

1. **装完系统就上线** — 默认配置=不安全，至少做完前 5 项再上线
2. **防火墙规则太多了认为万无一失** — 应用层漏洞防火墙管不了
3. **云服务器有安全组就不需要主机防火墙** — 纵深防御，两者都要有
4. **最小化安装后就不管了** — 新漏洞不断出现，需要持续维护

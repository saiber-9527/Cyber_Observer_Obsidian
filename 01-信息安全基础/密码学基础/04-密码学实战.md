# 密码学实战

> 真实场景下的密码学工具使用。

---

## OpenSSL 常用操作

### 生成 RSA 密钥对

```bash
# 生成私钥（2048位）
openssl genrsa -out private.pem 2048

# 提取公钥
openssl rsa -in private.pem -pubout -out public.pem

# 查看私钥详情
openssl rsa -in private.pem -text -noout
```

### 加密解密

```bash
# 用公钥加密文件
openssl pkeyutl -encrypt -inkey public.pem -pubin -in plain.txt -out cipher.bin

# 用私钥解密
openssl pkeyutl -decrypt -inkey private.pem -in cipher.bin -out decrypted.txt
```

### 数字签名

```bash
# 签名
openssl dgst -sha256 -sign private.pem -out sign.bin document.txt

# 验证签名
openssl dgst -sha256 -verify public.pem -signature sign.bin document.txt
```

### 生成自签名证书（HTTPS 测试用）

```bash
openssl req -x509 -newkey rsa:2048 -nodes \
  -keyout server.key -out server.crt \
  -days 365 -subj "/CN=localhost"
```

## GPG 文件加密

```bash
# 生成密钥对
gpg --full-generate-key

# 导出公钥
gpg --export -a "Your Name" > public.key

# 加密文件
gpg -e -r "Recipient Name" document.pdf

# 解密文件
gpg -d document.pdf.gpg

# 签名文件
gpg -s document.txt

# 签名+加密
gpg -se -r "Recipient Name" document.txt
```

## 哈希校验

```bash
# 计算文件 SHA-256
sha256sum file.iso

# 计算文件 MD5（不推荐，仅兼容旧系统）
md5sum file.iso

# 验证下载文件（比对校验和文件）
sha256sum -c checksums.txt
```

## SSH 密钥管理

```bash
# 生成 Ed25519 密钥对（推荐）
ssh-keygen -t ed25519 -C "your_email@example.com"

# 复制公钥到服务器
ssh-copy-id user@server.com

# 测试连接
ssh -i ~/.ssh/id_ed25519 user@server.com

# 配置多个密钥（~/.ssh/config）
# Host myserver
#   HostName 192.168.1.100
#   User admin
#   IdentityFile ~/.ssh/id_rsa_prod
```

## 常见陷阱

| 陷阱 | 正确做法 |
|------|---------|
| 用 ECB 模式加密 | 用 GCM 或 CBC |
| RSA 密钥长度 1024 | 至少 2048，推荐 4096 |
| 密码不经哈希直接存数据库 | 加盐后用 bcrypt/argon2 |
| 硬编码密钥在代码里 | 用环境变量/密钥管理服务 |
| 自签名证书用于生产 | 使用正规 CA 签发的证书 |

# 非对称加密

> **一对密钥**：公钥（Public Key）可以公开，私钥（Private Key）自己保留。
> 公钥加密的数据只有对应的私钥能解密。

---

## 核心特点

- ✅ **解决密钥分发问题**——公钥可以公开传递
- ❌ **慢**——不适合加密大量数据
- 核心用途：**密钥交换 + 数字签名**

---

## 常见算法

| 算法 | 密钥长度 | 安全强度 | 状态 |
|------|---------|---------|------|
| **RSA** | 2048 / 4096 bit | 够用 | ✅ 最广泛但正在被替换 |
| **ECC（椭圆曲线）** | 256 bit | ≈ RSA 3072 | ✅ 更高效，推荐 |
| **SM2** | 256 bit | 同 ECC | ✅ 国密标准 |
| **DSA** | — | 一般 | ❌ 不建议新系统使用 |
| **Ed25519** | 256 bit | 很高 | ✅ SSH、签名领域强力推荐 |

---

## RSA vs ECC

| 对比维度 | RSA | ECC |
|---------|-----|-----|
| 同等安全性下密钥长度 | 2048+ bit | 256 bit |
| 性能（签名/解密） | 慢 | 快 |
| 性能（验证/加密） | 快 | 稍慢 |
| 数学基础 | 大整数分解 | 椭圆曲线离散对数 |
| 趋势 | 逐步淘汰 | 成为新标准 |

## 典型应用

### 1. 密钥交换

```
场景：浏览器和服务器协商 HTTPS 密钥

客户端                           服务器
  │   "你好，我的非对称密钥是..."    │
  │─────────────────────────────→│
  │                                │
  │   用客户端公钥加密对称密钥      │
  │←─────────────────────────────│
  │                                │
  │  用私钥解密，拿到对称密钥       │
  │   之后用对称加密通信            │
```

### 2. 数字签名

- **签名者用私钥签名**，验证者用公钥验证
- 证明：消息确实来自声称的发送方
- 应用：代码签名、证书签发

### 3. SSH 认证

```
ssh-keygen -t ed25519   # 生成密钥对
ssh-copy-id user@host   # 公钥放到服务器
# 之后登录时用私钥证明身份
```

---

## ⚠️ 陷阱

1. **RSA 密钥太短** — 1024 bit 以下已被攻破，至少 2048 bit
2. **相同密钥多次使用** — 非对称加密一般只用于交换密钥（一次一密），不直接加密数据
3. **量子威胁** — Shor 算法可以破解 RSA/ECC，后量子密码学正在标准化中

# 身份与访问管理（IAM）

> 管好人（身份）和权限（访问），是安全的第一道门。

---

## IAM 三大组件

```
身份（Identity） ──→ 认证（AuthN） ──→ 授权（AuthZ）
    │                    │                  │
    谁是用户              证明你是谁          你能做什么
    ├ 员工                ├ 密码              ├ RBAC
    ├ 客户                ├ MFA               ├ ABAC
    ├ API 调用方          ├ SSO               ├ 权限策略
    └ 设备/服务           └ 生物特征          └ 属性
```

---

## 用户生命周期管理

```
入职 ──→ 分配身份 → 授予初始权限 → 启用账户
在职 ──→ 权限变更 → 定期审计 → 访问行为监控
离职 ──→ 冻结账户 → 回收权限 → 归档数据
```

**关键原则：** 入职即创建，离职即回收，变动即更新

---

## 企业 IAM 常见方案

| 类型 | 方案 | 说明 |
|------|------|------|
| **SSO 平台** | Okta、Azure AD、飞书 SSO | 一次登录访问所有系统 |
| **PAM（特权账号管理）** | 堡垒机、CyberArk | 管好管理员的高权限账号 |
| **MFA 方案** | TOTP、硬件 Key、生物识别 | 增加身份验证强度 |
| **CIAM（客户 IAM）** | Auth0、Keycloak | 面向外部用户（客户/合作伙伴） |

---

## ⚠️ 常见问题

1. **权限泛滥** — 员工换了部门，权限没跟着变
2. **共享账号** — 几个人用一个账号，出事找不到人
3. **僵尸账号** — 离职员工账号没回收，成为攻击入口
4. **特权账号不管理** — Admin/root 密码写死在脚本里
